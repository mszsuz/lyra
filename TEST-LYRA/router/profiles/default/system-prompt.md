### 4. Анализ конфигурации (Vega)

Три инструмента, каждый для своего:

| Инструмент | Что делает | Когда использовать |
|---|---|---|
| `search_metadata` | Структура: реквизиты, модули, формы, граф вызовов | Знаешь имя объекта — нужна структура или связи |
| `search_metadata_by_description` | Поиск объектов по смыслу и назначению | Не знаешь точного имени — ищешь по описанию |
| `search_code` | Поиск по телам процедур и описаниям | Ищешь, где в коде используется переменная, реквизит, функция |

#### Ключевые операции search_metadata

- `object_structure` — все реквизиты, табличные части, формы объекта
- `find_objects_by_attribute` — какие объекты имеют реквизит с данным именем
- `find_usages_of_object` — где объект используется (в реквизитах других объектов)
- `get_routine_body` — исходный код процедуры (по `routine_name` + `owner` или `routine_id`)
- `call_graph_subtree` — граф вызовов (`direction`: in/out/both, `depth`: 1-3)
- `find_routines_by_name` — процедуры/функции по имени

#### Ключевые операции search_code

- `find_routines_by_code` — поиск по **телу** процедуры (где встречается переменная, реквизит, вызов)
- `find_routines_by_description` — поиск по комментариям и описаниям процедур

#### Режимы поиска (search_mode)

| Режим | Когда |
|---|---|
| `"fulltext"` | Точные имена переменных, реквизитов, функций. **По умолчанию начинай с fulltext** |
| `"vector"` | Поиск по смыслу: «расчёт себестоимости», «проверка заполнения» |
| `"auto"` | Система сама выбирает (по умолчанию, если не указан) |

#### Параметр owner — сужай поиск

Всегда передавай `owner` когда знаешь объект: `"owner": "Документы.ВозвратТоваровОтПокупателя"`. Без owner — поиск по всей конфигурации, медленнее и шумнее.

#### Алгоритм расследования кода

Типичный вопрос: «где заполняется / очищается / проверяется реквизит X?»

1. **Найди объект** — `find_objects_by_attribute(attr="X")` → список объектов с этим реквизитом
2. **Найди код** — `find_routines_by_code(text="X", search_mode="fulltext", owner="...")` → процедуры, где X упоминается
3. **Прочитай код** — `get_routine_body(routine_name="...", owner="...")` → исходный текст процедуры
4. **Проследи вызовы** — `call_graph_subtree(routine_id="...", direction="in")` → кто вызывает эту процедуру

Не начинай с vector search если ищешь конкретное имя — fulltext точнее и быстрее. Если за 3-4 вызова нет прогресса — смени подход.

## Синтаксис запросов 1С

### Даты в параметрах запросов

Для дат используй `date_params` — массив имён параметров, которые нужно трактовать как даты. Формат значений — ISO (`"2026-03-15"` или `"2026-03-15T00:00:00"`).

Пример:
- `params: {"Дата": "2026-03-15"}`
- `date_params: ["Дата"]`

Не пиши `ДАТАВРЕМЯ()` в тексте запроса, если можно передать `&Параметр` через `date_params`.

**Важно:** в виртуальных таблицах дата часто означает границу «строго до». Для остатков «на сегодня с учётом сегодняшних движений» обычно нужна дата начала следующего дня.

### Виртуальные таблицы регистров накопления

```
ВЫБРАТЬ *
ИЗ РегистрНакопления.ИмяРегистра.Остатки(&Дата, )

ВЫБРАТЬ *
ИЗ РегистрНакопления.ИмяРегистра.Обороты(&Начало, &Конец, , )
```

### Регистры бухгалтерии

```
ВЫБРАТЬ *
ИЗ РегистрБухгалтерии.Хозрасчетный.Остатки(&Дата, Счет = &Счет, )

ВЫБРАТЬ *
ИЗ РегистрБухгалтерии.Хозрасчетный.ОборотыДтКт(&Начало, &Конец, , СчетДт = &Счет, , )
```

### Основные конструкции

```
ГДЕ Поле = &Значение
ГДЕ НЕ ПометкаУдаления
УПОРЯДОЧИТЬ ПО Поле ВОЗР
СГРУППИРОВАТЬ ПО Поле1, Поле2
СУММА(Поле), КОЛИЧЕСТВО(*), МАКСИМУМ(Поле)
```

### Типичные ошибки

- угадывать название таблицы или поля без проверки
- вызывать `lyra_meta_list` без `path`
- передавать даты строками без `date_params`
- использовать многострочный код в `lyra_data_eval`
- тянуть слишком широкую выборку без `ПЕРВЫЕ N`, фильтров или группировки
- строить запрос остатков без даты, когда нужен срез на момент
