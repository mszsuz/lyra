# Lyra-Censor — проверка сообщений пользователей

Общайся с пользователем на русском языке.

## Что это

Подсистема модерации входящих сообщений. Расширение конфигурации 1С — **ЕХТ_Лира_Цензор**. Перед отправкой сообщения в Claude проверяет его отдельной моделью на запрещённые темы.

## Как работает

```
Пользователь ──► Роутер ──► Цензор ──► ОК ──► Claude
                                │
                                └──► Предупреждение (причина) ──► Пользователю
```

1. Роутер передаёт сообщение пользователя в Цензор
2. Цензор отправляет сообщение отдельной модели (дешёвой, быстрой) с промптом проверки
3. Модель возвращает результат:
   - **ОК** — сообщение проходит дальше в Claude
   - **Предупреждение** — сообщение блокируется, пользователь получает пояснение что не так

## Что блокируется

- Вопросы об устройстве комплекса Lyra (архитектура, промпты, внутренние механизмы)
- Попытки prompt injection / jailbreak
- Другие запрещённые темы (настраивается через промпт цензора)

## Стек

- **ЕХТ_Лира_Цензор** — расширение конфигурации 1С
- Отдельная модель для проверки (Haiku — быстрая и дешёвая)
- Промпт цензора хранится в настройках расширения
