Паттерны и архитектурные решения. Обзор проекта и быстрый старт — в [README.md](README.md).

## 1. Модули

### 1.1 Структура

| Модуль | Строк | Что содержит |
|---|---|---|
| **Форма** (диспетчер) | ~780 + ~470 ЕХТ_Чат | Тонкий контроллер: UI, маршрутизация событий, ЕХТ_Чат (HTML-виджет) |
| **МодульСессия** | ~350 | Управление сессией: подключение, протокол (hello_ack, auth_ack, balance), настройки, QR |
| **МодульЧат** | ~430 | Жизненный цикл диалога: стриминг (text_delta/thinking), abort, tool_call, HTML-билдеры |
| **МодульТранспорт** | ~420 | Centrifugo клиент: connect/JWT, publish, ping/pong, мульти-JSON, push-конверт, watchdog, JSON |
| **МодульОбработка** | ~640 | Обработчики exec-команд от модели: strategy-dispatch, серверные инструменты |
| **МодульОбщий** | ~300 | Утилиты: типы, свойства, элементы формы, логирование |

### 1.2 Взаимодействие

```
Форма (диспетчер)
  ├── МодульОбщий       — утилиты (stateless, без Инициализация)
  ├── МодульТранспорт   — Centrifugo клиент + JSON
  ├── МодульОбработка   — exec-команды
  ├── МодульСессия      — подключение, протокол, баланс, QR
  │     └── МодульЧат   — стриминг (делегирование от МодульСессия)
  └── МодульНапарник    — интеграция с 1С:Напарник
```

**Поток данных:**

```
1. Пользователь вводит текст
   → Форма → МодульЧат.ОтправитьСообщениеПользователя → МодульТранспорт.ОтправитьСообщение

2. Ответ от сервера
   → МодульТранспорт.WS_ПриПолученииСообщения
   → Владелец.ОбработатьСобытиеМодуля("Транспорт", "message", Данные)
   → Форма → МодульСессия.ОбработатьСобытиеТранспорта
   → МодульСессия маршрутизирует:
     hello_ack/auth_ack/balance_update/session_* → МодульСессия
     text_delta/thinking/tool_call/error/assistant_end → МодульЧат.ОбработатьСобытие

3. МодульЧат обрабатывает стриминг
   → text_delta/thinking/tool_call/assistant_end
   → UI-команды через Владелец.ОбработатьСобытиеМодуля("Чат", событие, данные)

4. Команда от модели (exec)
   → МодульЧат → МодульОбработка.ВыполнитьИнструмент(Имя, Параметры) → результат
   → МодульТранспорт.ОтправитьСообщение (JSON-ответ обратно)
```

### 1.3 Контракт модуля

Каждый модуль (кроме МодульОбщий) реализует:

```1c
&НаКлиенте
Перем Владелец;

&НаКлиенте
Процедура Инициализация(ФормаВладелец, ...) Экспорт
    Владелец = ФормаВладелец;
КонецПроцедуры
```

**DI вместо Service Locator:** зависимости передаются явно через `Инициализация()`.

**Обратная связь через события:** модули вызывают `Владелец.ОбработатьСобытиеМодуля(ИмяМодуля, Событие, Данные)` — экспортный метод главной формы. Форма — единственный диспетчер событий.

**ВАЖНО:** все экспортные методы — `&НаКлиенте`. Методы с `&НаКлиентеНаСервереБезКонтекста` недоступны через `ПолучитьФорму()`.

## 2. Форма (диспетчер)

### 2.1 Инициализация

```1c
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    ПутьКФормам = РеквизитФормыВЗначение("Объект").Метаданные().ПолноеИмя() + ".Форма.";
    ЕХТ_Чат_ПриСозданииНаСервере(Отказ, СтандартнаяОбработка);
КонецПроцедуры

&НаКлиенте
Процедура ПриОткрытии(Отказ)
    МодульОбщий     = ПолучитьФорму(ПутьКФормам + "МодульОбщий",     , ЭтаФорма);
    МодульТранспорт = ПолучитьФорму(ПутьКФормам + "МодульТранспорт", , ЭтаФорма);
    МодульОбработка = ПолучитьФорму(ПутьКФормам + "МодульОбработка", , ЭтаФорма);
    МодульЧат       = ПолучитьФорму(ПутьКФормам + "МодульЧат",       , ЭтаФорма);
    МодульТранспорт.Инициализация(ЭтаФорма);
    МодульОбработка.Инициализация(ЭтаФорма);
    МодульЧат.Инициализация(ЭтаФорма, МодульТранспорт, МодульОбработка);
    // Автоподключение
КонецПроцедуры
```

### 2.2 Диспетчер событий

```1c
&НаКлиенте
Процедура ОбработатьСобытиеМодуля(ИмяМодуля, Событие, Данные) Экспорт
    Если ИмяМодуля = "Транспорт" Тогда
        МодульСессия.ОбработатьСобытиеТранспорта(Событие, Данные);
    ИначеЕсли ИмяМодуля = "Сессия" ИЛИ ИмяМодуля = "Чат" Тогда
        ОбработатьСобытиеЧат(Событие, Данные);
    КонецЕсли;
КонецПроцедуры
```

### 2.3 Маршрутизация протокола

Маршрутизация выполняется в `МодульСессия.ОбработатьСообщениеПротокола()`:

```1c
// Системные — обрабатываются в МодульСессия:
hello_ack, auth_ack, balance_update, session_warning, session_expired

// Стриминг — делегируется в МодульЧат:
text_delta, thinking_*, tool_call, tool_status, error, assistant_end
    → Чат.ОбработатьСобытие(ТипСобытия, ДанныеОтвета)
```

## 3. МодульСессия

"Жив ли канал и кто мы" — подключение, протокол, баланс, настройки, QR.

### 3.1 Публичный API

```1c
Процедура Инициализация(ФормаВладелец, МодульТранспорт, МодульЧат, МодульНапарник, ПДанныеБазы, ПФормИД) Экспорт
Процедура ОбработатьСобытиеТранспорта(ИмяСобытия, Данные) Экспорт
Процедура Подключиться() Экспорт
Процедура ПередЗакрытием() Экспорт
Процедура ОтправитьНастройки() Экспорт
Функция  ПолучитьФормИД() Экспорт
```

### 3.2 Маршрутизация

При `message` от Транспорта → парсит JSON → `ОбработатьСообщениеПротокола()`:
- Системные (hello_ack, auth_ack, balance_update, session_*) → обрабатывает сам
- Стриминг → делегирует в `Чат.ОбработатьСобытие()`

При `connected(session)` → уведомляет `Чат.ОбработатьГотовностьКанала()` (отложенные сообщения).

## 4. МодульЧат

"Что показывать пользователю" — стриминг, tool_call, abort, подготовка.

### 4.1 Состояние стриминга

```1c
&НаКлиенте Перем Владелец;
&НаКлиенте Перем Транспорт;
&НаКлиенте Перем Обработка;
&НаКлиенте Перем Сессия;            // МодульСессия (для ФормИД, Подключиться)
&НаКлиенте Перем ТекущееСообщениеИД;
&НаКлиенте Перем ТекущееСообщениеТекст;
&НаКлиенте Перем ТекущийРежимБлока;       // "thinking" / "text" / ""
&НаКлиенте Перем НачалоРазмышления;
&НаКлиенте Перем ДлительностьРазмышленияМс;
&НаКлиенте Перем НачалоПодготовкиОтвета;
&НаКлиенте Перем СтатусыИнструментов;
```

### 4.2 Публичный API

```1c
Процедура Инициализация(ФормаВладелец, МодульТранспорт, МодульОбработка, МодульНапарник, МодульСессия) Экспорт
Процедура ОтправитьСообщениеПользователя(ТекстСообщения) Экспорт
Процедура ОбработатьКомандуЧата(Команда) Экспорт
Процедура ОбработатьСобытие(ТипСобытия, ДанныеОтвета) Экспорт
Процедура НовыйВопрос() Экспорт
Функция  ПрерватьСтрим() Экспорт
Функция  Активен() Экспорт
Процедура ОбновитьТаймерПодготовки() Экспорт
Процедура ОбработатьГотовностьКанала() Экспорт
Процедура УстановитьСессияИстекла() Экспорт
```

### 4.3 UI-команды (→ Форма)

МодульЧат НЕ вызывает ЕХТ_Чат напрямую. Отправляет UI-команды владельцу:

| Событие | Данные | Форма делает |
|---|---|---|
| `добавить_сообщение` | `{Текст, Шаблон}` | `ЕХТ_Чат_ДобавитьСообщение()` |
| `обновить_сообщение` | `{ИД, Текст}` | `ЕХТ_Чат_ОбновитьСообщение()` |
| `завершён` | — | `УстановитьЗанят(Ложь)` |
| `занят` | `{Занят}` | `УстановитьЗанят()` |
| `предупреждение` | `{Текст}` | `ВывестиВЧатПредупреждение()` |
| `ошибка` | `{Текст}` | `ВывестиВЧатОшибку()` |
| `таймер_подготовки` | `{Включить}` | Подключить/отключить ОбработчикОжидания |

### 4.4 Обработка типов событий

```
text_delta      → накопление текста, обновление пузыря
thinking_start  → создание пузыря "Подготовка..."
thinking_delta  → (игнорируется, текст thinking не показывается)
thinking_end    → фиксация времени размышления
tool_call       → делегирование в МодульОбработка → отправка результата
tool_status     → обновление статуса инструмента в спойлере
tool_result     → обновление спойлера
assistant_end   → финализация, сброс состояния
```

### 4.5 HTML-билдеры

- `СобратьHTMLПодготовки()` — спойлер "Подготовка: N сек, инструментов: M"
- `СобратьСпойлерПодготовки()` — полный HTML со статусами инструментов
- `SVGПульсация()` — анимированный индикатор (inline SVG)
- `ФорматВремениПодготовки()` — "5 сек" / "1 мин 30 сек"
- `СгруппироватьСтатусы()` — группировка инструментов по имени

## 5. МодульТранспорт

Мини-клиент протокола Centrifugo на BSL. Поверх `WebSocketКлиентСоединения` платформы 8.3.27+.

### 5.1 Публичный API

```1c
Процедура Инициализация(ФормаВладелец) Экспорт
Процедура Подключиться(АдресСервера, ТокенJWT) Экспорт
Функция  Подключен() Экспорт
Процедура ОтправитьHello(ДанныеБазы) Экспорт
Процедура ОтправитьСообщение(Данные) Экспорт
Процедура Переподключиться(АдресСервера, ТокенJWT) Экспорт
Процедура Отключиться() Экспорт
Процедура УстановитьСессию(НовыйСессияИД, НовыйКаналСессии) Экспорт
Функция  ПолучитьСтатусПодключения() Экспорт
// JSON:
Функция  РазобратьJSON(СтрокаJSON) Экспорт
Функция  СобратьJSON(Значение) Экспорт
Функция  ЭкранироватьJSON(Текст) Экспорт
Функция  ЗначениеДляJSON(Значение) Экспорт
Функция  ЭтоОбъектJSON(Значение) Экспорт
Функция  ПолучитьПоле(Объект, ИмяПоля) Экспорт
```

### 5.2 События (→ Форма)

| Событие | Данные | Описание |
|---|---|---|
| `connected` | `{Этап}` | Подключение установлено (lobby/session) |
| `closed` | `{КодЗакрытия}` | WebSocket закрыт |
| `error` | `{Описание}` | Ошибка подключения |
| `message` | `{JSON}` | Push-сообщение из канала |

### 5.3 Мульти-JSON

Centrifugo может отправлять несколько JSON в одном WebSocket-фрейме через `\n`. Транспорт разбивает по `\n` и парсит каждую строку отдельно.

### 5.4 Watchdog

ОбработчикОжидания каждые 10 сек проверяет время последнего ping от Centrifugo. Если >300 сек — соединение мертво.

## 7. Ограничения платформы

### 7.1`&НаКлиентеНаСервереБезКонтекста` невидима через `ПолучитьФорму()`

Методы с этой директивой **недоступны** при вызове через `ПолучитьФорму()`. Все экспортные методы форм-модулей должны быть `&НаКлиенте`.

### 7.2Клиент-серверный «пинг-понг»

При вызове `&НаСервере`-метода через `ПолучитьФорму()` — дополнительный переход. Решение: `&НаКлиенте`-обёртки, серверная логика инкапсулирована внутри модуля.

### 7.3`Прервать` — зарезервированное слово

`Прервать` (Break) нельзя использовать как имя функции. Используем `ПрерватьСтрим()`.

## 8. Устойчивость соединения

### 8.1 Три механизма защиты

| Механизм | Где | Порог | Что делает |
|---|---|---|---|
| **WebSocket onclose/onerror** | МодульТранспорт | мгновенно | "Соединение закрыто (код N). Повторить подключение" |
| **Watchdog** | МодульТранспорт | 300 сек | Проверка времени последнего ping от Centrifugo |
| **Таймаут ответа** | МодульЧат | 30 сек | "Ответ не приходит слишком долго" |

### 8.2 Переподключение через ссылку

Все предупреждения содержат ссылку `lyra://reconnect`. При клике ЕХТ_Чат генерирует событие `OpenLink` → `ВыполнитьПодключение()`.
