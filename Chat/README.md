# Lyra-Chat-multiform.epf

> Модульная версия обработки Lyra-Chat. Монолитная форма (1521 строка) разнесена по служебным формам-модулям.
> Стриминг размышлений (thinking) и ответов Claude реализован в реальном времени.
> Паттерны взяты из анализа трёх production-обработок: СБИС (131 форма), Диадок (56 форм), Vanessa (108 форм).

- [Архитектура и паттерны](ARCHITECTURE.md) — модули, контракты, DI, события, ограничения платформы
- [Бэклог](BACKLOG.md) — статус реализации и план развития

## 1. Целевая архитектура

### Исходная версия (Lyra-Chat.epf)

```
Lyra-Chat.epf
├── Форма (1521 строка — god-object)
│   ├── UI-обработчики
│   ├── WebSocket/Bridge
│   ├── Парсинг JSON (строковый, хрупкий)
│   ├── MCP-инструменты (цепочка Если/ИначеЕсли)
│   ├── Вывод в чат
│   └── Копипаста ЕХТ_Чат (440 строк)
├── ФормаПодключение (пустая)
├── ObjectModule (пустой)
└── 2 макета
```

### Целевая архитектура (рефакторинг)

```
Lyra-Chat-multiform.epf
├── Форма (диспетчер)
│   ├── ОбработатьСобытиеМодуля() — единый маршрутизатор событий
│   ├── Владеет ЕХТ_Чат (HTML-виджет), но не формирует HTML
│   └── Минимум логики — только координация модулей и UI
├── МодульЧат — жизненный цикл диалога (стриминг, Стрим-структура, abort, пульсация)
├── МодульТранспорт — Centrifugo клиент целиком (WS, JWT, ping, hello, JSON)
├── МодульОбщий — утилиты для нескольких модулей (JSON, форматирование)
├── МодульОбработка — обработчики команд от модели (серверные и клиентские)
├── ObjectModule — общий серверный код, нужный нескольким модулям
└── 2 макета (HTML/JS чата)
```

**Паттерн:** Vanessa — главная форма хранит ссылки на модули в `Перем`, инициализирует их в `ПриОткрытии`, каскадно отключает в `ПередЗакрытием`. Серверный код живёт в том модуле, где используется (не всё в ObjectModule).

### Сравнение с оригиналом

| | Lyra-Chat.epf | Lyra-Chat-multiform.epf |
|---|---|---|
| Формы | 2 (Форма + пустая ФормаПодключение) | 5 (Форма + 3 модуля + ФормаПодключение) |
| Главная форма | 1521 строка (God-object) | 936 строк (контроллер + стриминг + ЕХТ_Чат) |
| JSON-парсинг | Строковый поиск через `СтрНайти` | `ЧтениеJSON` в Соответствие |
| MCP-dispatch | `Если/ИначеЕсли` цепочка | Реестр `Соответствие` (strategy) |
| WebSocket | Встроен в главную форму | Отдельный МодульBridge |
| Расширяемость | Только редактирование монолита | Добавление обработчиков через `ЗарегистрироватьОбработчик()` |

## 2. Сборка EPF

### 2.1 Конфигурация `.1c-devbase.bat`

Файл `.1c-devbase.bat` в корне проекта `Chat/` задаёт путь к платформе и базе для сборки:

```bat
@echo off
set "ONEC_PATH=C:\Program Files\1cv8\current\bin\1cv8.exe"
set "ONEC_FILEBASE_PATH=C:\1ext.ru\tools\1C-Services\TEMP\lyra-build"
```

**Важно:**
- Используется **файловая временная база** (не серверная) — для сборки .epf серверная база не нужна
- Каждый проект/агент должен использовать **свою папку** в `TEMP/`, т.к. базой могут пользоваться несколько агентов
- Путь **не должен содержать пробелов**
- Симлинк `C:\Program Files\1cv8\current\` — не хардкодить версию платформы

### 2.2 Создание временной базы

```bash
cmd //c "C:\\1ext.ru\\tools\\1C-Services\\DB\\create-temp-base.bat C:\\1ext.ru\\tools\\1C-Services\\TEMP\\lyra-build"
```

Временная база создаётся один раз и переиспользуется для всех сборок.

### 2.3 Сборка через `/1c-batch`

```bash
# Из папки Chat/ (где лежит .1c-devbase.bat):
build-epf.bat <XML-исходник> <Результирующий-EPF>
```

**Путь результата** — EPF должен собираться в корень папки EPF:

```
Chat/
├── .1c-devbase.bat
├── Lyra-Chat-multiform.epf/
│   ├── Lyra-Chat-multiform.epf    ← сюда собирается готовый файл
│   └── src/                        ← исходники XML/BSL
│       └── Lyra-Chat-multiform/
```

Пример вызова:

```bash
cd "C:/WORKS/2026-01-31 Lyra/Chat"
build-epf.bat "Lyra-Chat-multiform.epf/src" "Lyra-Chat-multiform.epf/Lyra-Chat-multiform.epf"
```

### 2.4 Запуск в веб-клиенте

1. Убедиться что Bridge запущен: `node C:/WORKS/2026-01-31 Lyra/Bridge/bridge.js`
2. Открыть обработку через URL веб-клиента:

```
http://localhost:8300/?C="Execute=C:\WORKS\2026-01-31 Lyra\Chat\Lyra-Chat-multiform.epf\Lyra-Chat-multiform.epf"
```

3. Нажать «Подключиться» — Bridge подключится к Claude
4. Ввести сообщение — увидеть стриминг thinking → text

### 2.5 Возможные проблемы

| Проблема | Причина | Решение |
|----------|---------|---------|
| Кириллица `1С-Services` ломается в cmd | Путь содержит символ `С` (кириллица) | Использовать латинскую `C`: `1C-Services` |
| `build-epf.bat` не находит `.1c-devbase.bat` | Текущая директория не `Chat/` | `cd` в `Chat/` или использовать bat-обёртку |
| Пути с пробелами ломают аргументы | Git Bash разбивает аргументы по пробелам | Использовать bat-обёртку с хардкодом путей |
| Сервисная база не отвечает после сборки | Designer блокирует базу при сборке | `srv1c-status.bat`, при необходимости `srv1c-start.bat` |

## 4. Как использовать

1. Создать временную базу (раздел 2.2, один раз)
2. Собрать EPF (раздел 2.3)
3. Запустить Bridge: `node bridge.js` из `Bridge/`
4. Открыть обработку в веб-клиенте (раздел 2.4)
5. Нажать «Подключиться» — проверить WebSocket через МодульBridge
6. Отправить сообщение — проверить стриминг thinking → text → result
7. Проверить MCP-запрос — проверить диспетч через МодульMCP
