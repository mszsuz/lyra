## 1. Архитектура и организация кода

### 1.1 Форма как «общий модуль»

В EPF нет общих модулей платформы. Все три обработки решают это одинаково: создают скрытые формы, которые работают как общие модули. Форма открывается через `ПолучитьФорму()` без показа пользователю, и её экспортные методы вызываются напрямую.

**Принцип:** форма-модуль не имеет видимых элементов, кэшируется в переменной главной формы, передача `ЭтаФорма` как владельца гарантирует автоматическое закрытие.

```
Источники: все три обработки
```

**Пример — Vanessa: объявление и инициализация модулей в главной форме**
```
Где: Форма "УправляемаяФорма", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\УправляемаяФорма\Ext\Form\Module.bsl
```
```1c
// Объявление модулей как переменных формы
#Область ГлобальныеПеременныеФормы

&НаКлиенте
Перем МодульРедакторТекста, МодульРаботаСБраузером, МодульРаботаСGit;
&НаКлиенте
Перем МодульОбщегоНазначенияVA;   // клиентский общий модуль
&НаКлиенте
Перем МодульСерверныеВызовы;      // серверный общий модуль
&НаКлиенте
Перем КэшФормПлагинов;           // плагины хранятся отдельно

#КонецОбласти
```

**Пример — СБИС: форма МодульОбъектаКлиент**
```
Где: Форма "МодульОбъектаКлиент", &НаКлиенте
Файл: СБИС\Sbis1C_UF.epf\src\Sbis1C_UF\Forms\МодульОбъектаКлиент\Ext\Form\Module.bsl
```
```1c
// Форма, которая возвращает саму себя — экспортные переменные = публичное состояние
&НаКлиенте
Перем ГлобальныйКэш Экспорт;

&НаКлиенте
Функция МодульОбъектаКлиент()
    Возврат ЭтаФорма;  // возвращает саму себя для использования как модуль
КонецФункции

&НаСервере
Функция МодульОбъектаСервер()
    Возврат РеквизитФормыВЗначение("Объект");  // доступ к серверному ObjectModule
КонецФункции
```

**Пример — Диадок: ленивая инициализация с синхронизацией контекста**
```
Где: Форма "Модуль_ПлатформаУправляемая", &НаКлиенте
Файл: Диадок\diadoc_4_62_0.epf\src\diadoc_4_62_0\Forms\Модуль_ПлатформаУправляемая\Ext\Form\Module.bsl
```
```1c
&НаКлиенте
Перем _МодульЯдроНаКлиенте;

&НаКлиенте
Функция Модуль_ЯдроНаКлиенте() Экспорт
    Если _МодульЯдроНаКлиенте <> Неопределено Тогда
        УстановитьКонтекстВызываемойФормы(_МодульЯдроНаКлиенте);  // синхронизация!
        Возврат _МодульЯдроНаКлиенте;
    КонецЕсли;
    ПараметрыФормы = Новый Структура;
    ПараметрыФормы.Вставить("ОбщийКонтекстКлиентСервер", Объект.ОбщийКонтекстКлиентСервер);
    _МодульЯдроНаКлиенте = ПолучитьФорму(ИмяФормыЯдро, ПараметрыФормы, ЭтаФорма);
    Возврат _МодульЯдроНаКлиенте;
КонецФункции
```

**Паттерны именования форм-модулей (сводка):**

| Префикс/Суффикс | Пример | Обработка |
|---|---|---|
| `Модуль_` | `Модуль_Платформа`, `Модуль_СинхрВызовы` | Диадок |
| `МодульОбъекта*` | `МодульОбъектаКлиент` | СБИС |
| `ОбщегоНазначения*` | `ОбщегоНазначенияVA` | Vanessa |
| `СерверныеВызовы` | `СерверныеВызовы` | Vanessa |
| `Плагин_` | `Плагин_УтвержденияBDD` | Vanessa |

**Архитектурная цена паттерна (ISP):** Форма-модуль формально нарушает принцип разделения интерфейсов (Interface Segregation Principle) — вызывающий код получает весь интерфейс `УправляемаяФорма` (сотни свойств и методов), хотя использует лишь 3–5 экспортных методов модуля. **Однако в EPF это единственный механизм модуляризации**, и при соблюдении конвенций (минимальный экспортный интерфейс, документирование публичных методов) он надёжно работает во всех трёх обработках. Альтернативы — макеты с `Выполнить()` — значительно хуже: теряется автодополнение, статический анализ и навигация по коду.

### 1.2 Service Locator / Кэш-контекст

СБИС использует структуру-контекст `Кэш`, которая передаётся первым параметром во все функции. Это реализация паттерна Service Locator — единая точка доступа к сервисам, настройкам и состоянию.

```
Где: Форма "МодульОбъектаКлиент", &НаКлиенте
Файл: СБИС\Sbis1C_UF.epf\src\Sbis1C_UF\Forms\МодульОбъектаКлиент\Ext\Form\Module.bsl
Строки: 24127-24172
```
```1c
// Конструктор кэш-контекста — создаёт структуру со всеми сервисами
Функция НовыйЛокальныйКэш_Пустой()
    НовыйКэш = Новый Структура(
        "_Класс, ТихийРежим, СБИС, ТекущийСеанс, Парам, ВладелецФормы, ГлавноеОкно",
        "ЛокальныйКэш"  // поле _Класс — идентификатор "типа" структуры
    );
    Возврат НовыйКэш;
КонецФункции
```

**Лайфхак:** структура `Кэш` заменяет глобальные переменные — любая функция получает доступ к нужным сервисам через единый параметр. Это упрощает тестирование (можно подменить содержимое) и делает зависимости явными.

### 1.3 Плагинная архитектура

Vanessa реализует полноценную плагинную систему с формальным контрактом. Каждый плагин — это форма, реализующая три обязательных метода.

**Контракт плагина:**

```
Где: ObjectModule
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Ext\ObjectModule.bsl
Строки: 12021-12034
```
```1c
// Серверная часть контракта — описание плагина
Функция ОписаниеПлагина(ВозможныеТипыПлагинов) Экспорт
    Результат = Новый Структура;
    Результат.Вставить("Тип", ВозможныеТипыПлагинов.Утилита);
    Результат.Вставить("Идентификатор", Метаданные().Имя);
    Результат.Вставить("Представление", "ЗапросыИзБД");
    Возврат Новый ФиксированнаяСтруктура(Результат);
КонецФункции

Процедура Инициализация(КонтекстЯдраПараметр) Экспорт
    ТекстСообщенийПользователю = КонтекстЯдраПараметр.ТекстСообщенийПользователю;
КонецПроцедуры
```

**Клиентская часть — обязательный контракт формы-плагина:**

```
Где: Форма "Плагин_*", &НаКлиенте
```
```1c
// 1. Инициализация — получение контекста фреймворка
&НаКлиенте
Процедура УстановитьСлужебныеПеременные(КонтекстФреймворка) Экспорт
    Ванесса = КонтекстФреймворка;
КонецПроцедуры

// 2. Освобождение ресурсов
&НаКлиенте
Функция ОтключениеМодуля() Экспорт
    Ванесса = Неопределено;
КонецФункции

// 3. Описание возможностей
&НаКлиенте
Функция ОписаниеПлагина(ВозможныеТипыПлагинов) Экспорт
    Возврат ОписаниеПлагинаНаСервере(ВозможныеТипыПлагинов);
КонецФункции
```

**Диадок** тоже использует систему зависимостей-модулей, но по-своему — через конструктор зависимостей:

```
Где: ObjectModule
Файл: Диадок\diadoc_4_62_0.epf\src\diadoc_4_62_0\Ext\ObjectModule.bsl
Строки: 3191-3244
```
```1c
// Конструктор зависимости — выбирает источник в зависимости от режима отладки
Функция Новый_Зависимость(Имя, ИмяМакета)
    Если ЭтоВнешняяОбработка() Тогда
        Тип = "ВнешняяОбработкаОбъект";
    Иначе
        Тип = "ОбработкаОбъект";
    КонецЕсли;
    Если РежимОтладкиВключен() Тогда
        СхемаИсточника = "file://";               // отладка: загрузка из файла
        Источник = СхемаИсточника + ПутьКФайлу(ИмяМакета) + ".epf";
    Иначе
        СхемаИсточника = "template://";            // продакшн: из макета
        Источник = СхемаИсточника + ИмяМакета;
    КонецЕсли;
    Результат = Контракт_Зависимость();
    Результат.Имя = "КонтурДиадок" + Имя;
    Результат.Тип = Тип;
    Результат.Источник = Источник;
    Возврат Результат;
КонецФункции
```

**Лайфхак:** плагинный контракт из 3 методов (`Инициализация`, `ОтключениеМодуля`, `ОписаниеПлагина`) позволяет добавлять новую функциональность без изменения ядра.

### 1.4 Strategy + Fallback диспетчеризация

СБИС ищет специализированную форму по имени, а если не находит — использует базовый шаблон. Это позволяет добавлять новые типы документов просто создавая форму с нужным именем.

```
Где: Форма "МодульОбъектаКлиент", &НаКлиенте
Файл: СБИС\Sbis1C_UF.epf\src\Sbis1C_UF\Forms\МодульОбъектаКлиент\Ext\Form\Module.bsl
```
```1c
// Сначала ищем специализированную форму, потом откатываемся к базовой
фрм = ГлавноеОкно.сбисНайтиФормуФункции(
    "ПолучитьТабличнуюЧастьДокумента1С",  // имя метода
    "Файл_" + ТипДокумента,               // специализированная форма
    "Файл_Шаблон",                         // запасная базовая форма
    Кэш
);
фрм.ПолучитьТабличнуюЧастьДокумента1С(Кэш, Контекст);
```

**Vanessa** использует аналогичный паттерн — выбор формы по `ВерсияПоставки`:

```
Где: Форма "УправляемаяФорма", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\УправляемаяФорма\Ext\Form\Module.bsl
Строки: 29670-29704
```
```1c
// Strategy: выбор формы по ВерсияПоставки
Если Объект.ВерсияПоставки = "single" Тогда
    // Встроенная форма — single-поставка
    МодульРаботыСВидео = ПолучитьФорму(
        ПолучитьИмяОбработкиVA() + ".Форма.Video_РаботаСВидео", ПараметрыФормы);
Иначе
    // Fallback: внешняя обработка из файловой системы
    ИмяФайла = ДополнитьСлешВПуть(Объект.КаталогИнструментов)
        + "lib/Video/РаботаСВидео.epf";
    МодульРаботыСВидео = ПолучитьФорму(
        "ВнешняяОбработка.РаботаСВидео.Форма.Форма");
КонецЕсли;
```

**Лайфхак:** шаблон `Файл_ТипДокумента → Файл_Шаблон` работает как реестр обработчиков — добавляешь форму с нужным именем и она автоматически подхватывается.

### 1.5 Роль ObjectModule

Все три обработки используют ObjectModule как серверный контейнер состояния и бизнес-логики. Переменные организованы областями по компонентам.

```
Где: ObjectModule
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Ext\ObjectModule.bsl
```
```1c
// Переменные разделены по компонентам через области
#Область ПеременныеFeatureReader
Перем ТаблицаКлючевыхСлов;
Перем ЭтоУФ Экспорт;
#КонецОбласти

#Область ПеременныеЗапросыИзБД
// ...
#КонецОбласти

#Область ПеременныеУтвержденияBDD
Перем ФлагОтрицанияДляСообщения;
#КонецОбласти
```

**Правило разделения:**

| Что | Где |
|---|---|
| Серверное состояние, переживающее несколько вызовов | ObjectModule |
| Клиентское состояние, кэши, ссылки на модули | Переменные формы |
| Код, общий для клиента и сервера | `&НаКлиентеНаСервереБезКонтекста` |

### 1.6 Include-система (сборка из библиотек)

СБИС размечает код областями-маркерами `#Область include_*`, что указывает на сборку из отдельных файлов-библиотек.

```
Где: ObjectModule
Файл: СБИС\Sbis1C_UF.epf\src\Sbis1C_UF\Ext\ObjectModule.bsl
```
```1c
#Область include_core_base_Helpers_ModuleGetters
    // Код из библиотеки core/base/Helpers/ModuleGetters
#КонецОбласти

#Область include_core2_vo2_МодульОбъекта_АПИ_Клиент
    // Код из библиотеки core2/vo2/МодульОбъекта/АПИ/Клиент
#КонецОбласти
```

**Важно:** это признак системы сборки, а не стиль программирования. Для обработки с 10–30 формами этот подход избыточен, но сама идея разделения кода областями по компонентам — полезна.

### 1.7 Fluent API через возврат ЭтаФорма

Плагины Vanessa возвращают `ЭтаФорма` из каждого метода, что позволяет строить цепочки вызовов в стиле Fluent API.

```
Где: Форма "Плагин_УтвержденияBDD", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\Плагин_УтвержденияBDD\Ext\Form\Module.bsl
```
```1c
// Использование в клиентском коде:
Утверждения.Что(ФактическоеЗначение).Не_().Равно(ОжидаемоеЗначение).ЭтоИстина();

// Реализация:
&НаКлиенте
Функция Что(Знач Значение, Знач Сообщение = "") Экспорт
    ВыполнитьМетодОбъектаНаСервере(...);
    Возврат ЭтаФорма;  // ключ паттерна — возврат self
КонецФункции

&НаКлиенте
Функция Не_() Экспорт
    ВыполнитьМетодОбъектаНаСервере(...);
    Возврат ЭтаФорма;
КонецФункции
```

**Лайфхак:** возврат `ЭтаФорма` из методов формы-модуля создаёт Fluent-интерфейс. Удобно для Builder-паттернов при конструировании запросов или конфигураций.

## 2. Инициализация и жизненный цикл

### 2.1 Асинхронная инициализация модулей

Vanessa инициализирует модули в `ПриОткрытии`, а не в `ПриСозданииНаСервере`, чтобы разделить серверную и клиентскую инициализацию.

```
Где: Форма "УправляемаяФорма", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\УправляемаяФорма\Ext\Form\Module.bsl
```
```1c
&НаКлиенте
Процедура ПриОткрытии(Отказ)
    // Модули создаются на клиенте после открытия формы
    МодульОбщегоНазначенияVA = ПолучитьФорму(
        ПолучитьИмяОбработкиVA() + ".Форма.ОбщегоНазначенияVA", , ЭтаФорма);
    МодульОбщегоНазначенияVA.ИнициализацияФормы(ЭтаФорма);

    МодульСерверныеВызовы = ПолучитьФорму(
        ПолучитьИмяОбработкиVA() + ".Форма.СерверныеВызовы", , ЭтаФорма);
КонецПроцедуры
```

**Правило:** `ПриСозданииНаСервере` — для серверной инициализации (загрузка макетов, чтение настроек). `ПриОткрытии` — для клиентской (создание модулей, подключение обработчиков).

### 2.2 Идемпотентная инициализация (ленивая загрузка)

Диадок загружает модули только при первом обращении. Повторный вызов возвращает кэш.

```
Где: Форма "Модуль_ПлатформаУправляемая", &НаКлиенте
Файл: Диадок\diadoc_4_62_0.epf\src\diadoc_4_62_0\Forms\Модуль_ПлатформаУправляемая\Ext\Form\Module.bsl
```

```1c
    // Идемпотентность: если уже загружен — просто возвращаем
    Если _МодульЯдроНаКлиенте <> Неопределено Тогда

```

### 2.3 Трёхуровневый кэш плагинов (Неопределено / Истина / Объект)

Vanessa использует три состояния для кэша плагинов: `Неопределено` (неизвестный), `Истина` (зарегистрирован, но не загружен), объект формы (загружен и готов).

```
Где: Форма "УправляемаяФорма", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\УправляемаяФорма\Ext\Form\Module.bsl
Строки: 17614-17649
```
```1c
&НаКлиенте
Функция Плагин(Знач Идентификатор) Экспорт
    ОбъектПлагин = Неопределено;
    Если КэшФормПлагинов.Свойство(Идентификатор) Тогда
        ОбъектПлагин = КэшФормПлагинов[Идентификатор];
    КонецЕсли;

    Если ОбъектПлагин = Неопределено Тогда
        ВызватьИсключение "Плагин не зарегистрирован: " + Идентификатор;

    ИначеЕсли ОбъектПлагин = Истина Тогда
        // Lazy load: Истина означает «зарегистрирован, но не загружен»
        ИмяПлагина = Объект.Плагины[Идентификатор];
        Если Объект.ВерсияПоставки = "single" Тогда
            ИмяФормы = ПолучитьИмяОбработкиVA() + ".Форма.Плагин_" + ИмяПлагина;
        Иначе
            ИмяФормы = "ВнешняяОбработка." + ИмяПлагина + ".Форма";
        КонецЕсли;
        ОбъектПлагин = ПолучитьФорму(ИмяФормы, , ЭтаФорма);
        ОбъектПлагин.УстановитьСлужебныеПеременные(ЭтаФорма);
        МодулиДляОтключения.Добавить(ОбъектПлагин);
        КэшФормПлагинов.Вставить(Идентификатор, ОбъектПлагин);
    КонецЕсли;

    Возврат ОбъектПлагин;
КонецФункции
```

**Лайфхак:** трёхуровневый кэш позволяет отделить регистрацию от загрузки. Плагины регистрируются быстро (Истина), а дорогое `ПолучитьФорму` происходит только при первом использовании.

## 3. Взаимодействие между формами

### 3.1 ОткрытьФорму vs ПолучитьФорму

Два основных способа работы с формами:

| Метод | Когда использовать | Форма видна? |
|---|---|---|
| `ОткрытьФорму()` | UI-формы, диалоги, формы с пользовательским интерфейсом | Да |
| `ПолучитьФорму()` | Формы-модули, служебные формы, формы-утилиты | Нет |

```1c
// ОткрытьФорму — для UI-форм
ПараметрыФормы = Новый Структура;
ПараметрыФормы.Вставить("Заголовок", "Моя форма");
ОткрытьФорму("ВнешняяОбработка.МояОбработка.Форма.МояПодФорма",
    ПараметрыФормы,
    ЭтаФорма,   // владелец
    ,,,,
    Новый ОписаниеОповещения("ОбработчикЗакрытия", ЭтаФорма)
);

// ПолучитьФорму — для модулей (невидимые)
МодульСервер = ПолучитьФорму(
    "ВнешняяОбработка.МояОбработка.Форма.СерверныеВызовы",
    Новый Структура,
    ЭтаФорма  // владелец
);
```

**Нюанс производительности (клиент-серверный «пинг-понг»):** При вызове `&НаСервере`-метода формы-модуля через `ПолучитьФорму()` происходит дополнительный клиент-серверный переход. Если главная форма уже на сервере и вызывает серверный метод модуля — получается цепочка клиент → сервер (главная форма) → клиент → сервер (форма-модуль). **Решение**, применённое во всех трёх обработках: для межформенных вызовов использовать `&НаКлиенте`-обёртки (см. паттерн Vanessa в 4.1), а серверную логику инкапсулировать внутри формы-модуля. Это не отменяет ценности паттерна — просто требует осознанного выбора директив компиляции.

### 3.5 Оповещения с фильтрацией по UID

И Диадок, и Vanessa фильтруют глобальные оповещения по уникальному идентификатору формы, чтобы одновременно открытые экземпляры не мешали друг другу.

**Пример — Vanessa:**
```
Где: Форма "УправляемаяФорма", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\УправляемаяФорма\Ext\Form\Module.bsl
Строки: 1093-1134
```
```1c
&НаКлиенте
Процедура ОбработкаОповещения(ИмяСобытия, Параметр, Источник)
    // Guard: форма закрыта — игнорируем
    Если НЕ ЭтаФорма.Открыта() И НЕ ВыполнениеСценариевБезОткрытияФормы Тогда
        Возврат;
    КонецЕсли;

    Если ИмяСобытия = "ВыборИзвестногоШага" Тогда
        // Фильтрация по UID: обрабатываем только «свои» оповещения
        Если Параметр.ИдФормы = ЭтаФорма.УникальныйИдентификатор Тогда
            МодульРедакторТекста().ВставитьТекст(Параметр.Текст);
        КонецЕсли;

    ИначеЕсли ИмяСобытия = "РедактированиеТаблицыGherkin" Тогда
        Если Параметр.UID <> УникальныйИдентификатор Тогда
            Возврат;  // не наше оповещение
        КонецЕсли;
        ЗаменитьТаблицуGherkinПослеРедактирования(Параметр.МассивСтрок);
    КонецЕсли;
КонецПроцедуры
```

**Пример — Диадок:**
```
Где: Форма "ФормаУправляемая", &НаКлиенте
Файл: Диадок\diadoc_4_62_0.epf\src\diadoc_4_62_0\Forms\ФормаУправляемая\Ext\Form\Module.bsl
Строки: 1342-1373
```
```1c
&НаКлиенте
Процедура ОбработкаОповещения(ИмяСобытия, Параметр, Источник)
    ИсточникОповещения = ИдентификаторОсновнойФормы();

    // Фильтрация: игнорируем чужие оповещения
    Если Не ЭтаФорма.Открыта() ИЛИ ИсточникОповещения <> Источник Тогда
        Возврат;
    КонецЕсли;

    Если ИмяСобытия = "Диадок_ОбновитьГлавныйСписок" Тогда
        ОбновитьСписок(НовыйРежим, Параметр);
    КонецЕсли;
КонецПроцедуры
```

**Лайфхак:** при отправке оповещения всегда включайте UID формы-источника. При получении — проверяйте UID перед обработкой. Это предотвращает конфликты при множественных экземплярах.

### 3.6 Каскадное закрытие модулей

Vanessa хранит список загруженных модулей и отключает их при закрытии формы.

```
Где: Форма "УправляемаяФорма", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\УправляемаяФорма\Ext\Form\Module.bsl
Строки: 1326-1358
```
```1c
&НаКлиенте
Процедура ВыполнитьОтключениеМодулей(ПроисходитЗавершениеРаботы)
    // При завершении работы платформы — не отключаем (бесполезно)
    Если ПроисходитЗавершениеРаботы = Истина Тогда
        Возврат;
    КонецЕсли;

    // Отключаем основные модули
    Для Каждого ТекМодуль Из МодулиДляОтключения Цикл
        ТекМодуль.ОтключениеМодуля();
    КонецЦикла;
    МодулиДляОтключения.Очистить();

    // Отключаем внешние обработки
    Для Каждого Строка Из ТаблицаКонтекстовОбработок Цикл
        Если Строка.Обработка = Неопределено Тогда
            Продолжить;
        КонецЕсли;
        Попытка
            Строка.Обработка.ОтключениеМодуля();
        Исключение
            // Глотаем ошибки при отключении — главное закрыться
        КонецПопытки;
    КонецЦикла;
КонецПроцедуры
```

## 4. Клиент-серверное взаимодействие

### 4.1 Серверный прокси через форму СерверныеВызовы

Vanessa выделяет отдельную форму для серверных вызовов. Клиентский метод с `Экспорт` — публичный API, серверный без `Экспорт` — реализация.

```
Где: Форма "СерверныеВызовы", &НаКлиенте + &НаСервере
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\СерверныеВызовы\Ext\Form\Module.bsl
```
```1c
// Доступ к объекту обработки на сервере
&НаСервере
Функция Объект() Экспорт
    Возврат РеквизитФормыВЗначение("Объект");
КонецФункции

// Паттерн: клиентский фасад → серверная реализация
&НаКлиенте
Процедура ЯВыполняюКодВстроенногоЯзыкаНаСервере(Текст) Экспорт
    ЯВыполняюКодВстроенногоЯзыкаНаСервереСервер(Текст);
КонецПроцедуры

&НаСервере
Процедура ЯВыполняюКодВстроенногоЯзыкаНаСервереСервер(Текст)
    Объект().МетодОбъекта(Текст);  // вызов метода ObjectModule
КонецПроцедуры
```

**Правило:** клиентский метод с суффиксом `Экспорт` — публичный API модуля. Серверный без `Экспорт` — внутренняя реализация. Это чётко разделяет контракт и детали.

### 4.2 НаКлиентеНаСервереБезКонтекста — общий код

Директива `&НаКлиентеНаСервереБезКонтекста` позволяет писать код, который работает и на клиенте, и на сервере. Ограничение: нет доступа к реквизитам формы, поэтому форма передаётся параметром.

**Пример — Диадок: управление видимостью элементов**
```
Где: Форма "ФормаЭлементаСправочникаУправляемая", &НаКлиентеНаСервереБезКонтекста
Файл: Диадок\diadoc_4_62_0.epf\src\diadoc_4_62_0\Forms\ФормаЭлементаСправочникаУправляемая\Ext\Form\Module.bsl
Строки: 407-426
```
```1c
&НаКлиентеНаСервереБезКонтекста
Процедура УстановитьВидимостьПредупреждения(Форма, ПоказыватьПриОтсутствииСвязей = Истина)
    // Форма передаётся параметром — нет контекста
    ГруппаСтраницы = Форма.Элементы.ГруппаСтраницы;
    ТекущаяСтраница = ГруппаСтраницы.ПодчиненныеЭлементы.СтраницаТаблица;

    ЭтоФормаКонтрагента = (Форма.ИмяСправочника = "Контрагенты");
    Если ЭтоФормаКонтрагента Тогда
        НетСвязанных = (Форма.ТаблицаСвязанныеСправочники.Количество() = 0);
        Если ПоказыватьПриОтсутствииСвязей И НетСвязанных Тогда
            ТекущаяСтраница = ГруппаСтраницы.ПодчиненныеЭлементы.СтраницаПредупреждение;
        КонецЕсли;
    КонецЕсли;

    ГруппаСтраницы.ТекущаяСтраница = ТекущаяСтраница;
КонецПроцедуры
```

**Пример — Диадок: динамическое переключение типа и видимости поля**
```
Где: Форма "НастройкиУправляемая", &НаКлиентеНаСервереБезКонтекста
Файл: Диадок\diadoc_4_62_0.epf\src\diadoc_4_62_0\Forms\НастройкиУправляемая\Ext\Form\Module.bsl
Строки: 925-961
```
```1c
&НаКлиентеНаСервереБезКонтекста
Процедура УстановитьСвойстваПоляПодключаемыйМодульПуть(Форма, РасположениеПМ)
    Если РасположениеПМ = АдресПользовательскогоМодуляВнешниеОбработки() Тогда
        ТипЭлемента = ОписаниеТипаСправочникаДополнительныхОбработок();
        ЗаголовокЭлемента = "Доп. обработка";
    ИначеЕсли РасположениеПМ = АдресПользовательскогоМодуляКонфигурация() Тогда
        ТипЭлемента = ОписаниеТипаСтрока();
        ЗаголовокЭлемента = "Имя обработки";
    Иначе
        ТипЭлемента = ОписаниеТипаНеопределено();
        ЗаголовокЭлемента = "<Неприменимо>";
    КонецЕсли;

    ЭлементПоля = Форма.Элементы.ПодключаемыйМодульПуть;
    ЭлементПоля.ОграничениеТипа = ТипЭлемента;
    ЭлементПоля.Заголовок = ЗаголовокЭлемента;
    Форма.ПодключаемыйМодульПуть = ТипЭлемента.ПривестиЗначение(Форма.ПодключаемыйМодульПуть);
КонецПроцедуры
```

**Лайфхак:** `&НаКлиентеНаСервереБезКонтекста` идеален для функций управления UI, валидации, форматирования — всего, что нужно и при открытии формы на сервере, и при интерактивной работе на клиенте.

**Ограничение при межформенных вызовах:** Методы с `&НаКлиентеНаСервереБезКонтекста` **недоступны** при вызове через `ПолучитьФорму()`. Платформа не находит такие методы у полученной формы — ошибка: `Метод объекта не обнаружен (ИмяМетода)`. Через `ПолучитьФорму()` видны только методы с директивами `&НаКлиенте` и `&НаСервере`. Это важно для паттерна «форма как модуль»: все экспортные методы форм-модулей, вызываемые из других форм, должны быть `&НаКлиенте`, а не `&НаКлиентеНаСервереБезКонтекста`. Директива `&НаКлиентеНаСервереБезКонтекста` работает только для вызовов **внутри той же формы**.

## 5. Кэширование

### 5.1 Кэш в переменных формы

Стандартный приём во всех трёх обработках — кэширование дорогих вычислений в переменных формы.

```
Где: Форма "ОбщегоНазначенияVA", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\ОбщегоНазначенияVA\Ext\Form\Module.bsl
```

```1c
    Если КешПолучитьСтруктуруПараметров = Неопределено Тогда
        // Первый вызов — вычисляем и кэшируем
```

### 5.2 Lazy-init кэш на Соответствии

Диадок реализует key-value кэш в ObjectModule через `Соответствие` с ленивой инициализацией.

```
Где: ObjectModule
Файл: Диадок\diadoc_4_62_0.epf\src\diadoc_4_62_0\Ext\ObjectModule.bsl
Строки: 1-4 (объявление), 1296-1326 (реализация)
```
```1c
// Объявление (строка 4):
Перем Кэш Экспорт;

// Чтение: если кэш не создан — просто Неопределено
Функция Кэш_Прочитать(ИмяЗначения)
    Результат = Неопределено;
    Если Кэш <> Неопределено Тогда
        Результат = Кэш.Получить(ИмяЗначения);
    КонецЕсли;
    Возврат Результат;
КонецФункции

// Запись: кэш создаётся при первой записи
Процедура Кэш_Поместить(ИмяЗначения, Значение)
    Если Кэш = Неопределено Тогда
        Кэш = Новый Соответствие;  // ленивая инициализация
    КонецЕсли;
    Кэш.Вставить(ИмяЗначения, Значение);
КонецПроцедуры

// Полная очистка
Процедура Кэш_Очистить() Экспорт
    Если Кэш <> Неопределено Тогда
        Кэш.Очистить();
    КонецЕсли;
КонецПроцедуры
```

**Лайфхак:** `Соответствие` вместо `Структура` для кэша — не нужно заранее знать ключи. Lazy-init (`Если = Неопределено Тогда Новый`) экономит память, если кэш не понадобится.

```1c
    Ошибка = ИнформацияОбОшибке();
    // Все ошибки идут в ядро — единый обработчик
    Ядро.Ошибка_Обработать(ВидОперации, Ошибка, ТекстСообщения);
    Возврат;
КонецПопытки;
```

**Масштаб и риски централизации:** Ядро Диадока достигает десятков тысяч строк кода, что делает его God-object'ом. При этом Ядро зависит от хранилищ, а хранилища — от Ядра (циклические зависимости). Хранилища содержат не только CRUD-операции, но и бизнес-логику (смешение ответственностей). **Однако** сам принцип централизованной обработки ошибок через выделенный модуль — правильный; проблема в масштабе, а не в подходе. **Для новых обработок:** рекомендуется ограничивать размер «ядра» и выносить бизнес-логику в отдельные модули с явными контрактами.

## 11. Безопасность

### 11.1 XOR-обфускация паролей

СБИС шифрует пароли XOR с маркером-префиксом `#ЁЪ#0` для определения, зашифрован ли пароль.

```
Где: ObjectModule
Файл: СБИС\Sbis1C_UF.epf\src\Sbis1C_UF\Ext\ObjectModule.bsl
Строки: 67407-67475
```
```1c
Функция ШифроватьXOR(Текст) Экспорт
    Ключ = "wLZhP7CibDtFKTi9bLHkcQSLzYRGVlmW";
    Результат = "";
    Для i = 1 По СтрДлина(Текст) Цикл
        ПозицияКлюча = (i - 1) % СтрДлина(Ключ) + 1;
        КодТекста = КодСимвола(Сред(Текст, i, 1));
        КодКлюча = КодСимвола(Сред(Ключ, ПозицияКлюча, 1));
        Результат = Результат + Символ(КодТекста + КодКлюча);
    КонецЦикла;
    Возврат "#ЁЪ#0" + Результат;  // маркер-префикс
КонецФункции

Функция РасшифроватьXOR(Знач Текст, context_params = Неопределено)
    Если НЕ Лев(Текст, 5) = "#ЁЪ#0" Тогда
        // Текст не зашифрован — зашифровать и сохранить
        ЗашифрованныйТекст = ШифроватьXOR(Текст);
        context_params.password = ЗашифрованныйТекст;
        Возврат Текст;  // вернуть оригинал
    КонецЕсли;
    // Расшифровать: обратная операция (- вместо +)
    ЗашифрованныйТекст = Сред(Текст, 6);
    // ... КодСимвола - КодКлюча ...
КонецФункции
```

**Важно:** это не криптография, а обфускация — защита от случайного чтения пароля из настроек. Маркер `#ЁЪ#0` позволяет автоматически определить, нужна ли расшифровка.

### 13.2 Каскадный поиск макетов (Chain of Responsibility)

Vanessa ищет макет последовательно в нескольких местах — от специализированного к общему.

```
Где: Форма "УправляемаяФорма", &НаКлиенте
Файл: vanessa-automation-single.epf\src\vanessa-automation-single\Forms\УправляемаяФорма\Ext\Form\Module.bsl
Строки: 12810-12893
```
```1c
&НаКлиенте
Функция ПолучитьМакетОбработки(ИмяМакета, ДопПараметры = Неопределено) Экспорт

    // 1. В обработке подчинённого сценария
    Обработка = ПолучитьОбработкаСвязаннаяСИсполняемойФичей(ИмяФичи);
    Если Обработка <> Неопределено Тогда
        Макет = Обработка.ПолучитьМакетОбработки(ИмяМакета);
        Если Макет <> Неопределено Тогда Возврат Макет; КонецЕсли;
    КонецЕсли;

    // 2. В обработке основной фичи
    Если СтрокаСценария.ФичаИмеетСвоюEPF Тогда
        Макет = ПолучитьМакетИзОбработкиФичи(ИмяМакета);
        Если Макет <> Неопределено Тогда Возврат Макет; КонецЕсли;
    КонецЕсли;

    // 3. Как встроенный макет (single-поставка)
    Если Объект.ВерсияПоставки = "single" Тогда
        Макет = ПолучитьВстроенныйМакетПоИмени(ИмяМакета, ИмяФичи);
        Если Макет <> Неопределено Тогда Возврат Макет; КонецЕсли;
    КонецЕсли;

    // 4. В каталоге проекта
    Если ЗначениеЗаполнено(Объект.КаталогПроекта) Тогда
        Макет = НайтиМакетВКаталогеПроекта(ИмяМакета, ДопПараметры);
        Если Макет <> Неопределено Тогда Возврат Макет; КонецЕсли;
    КонецЕсли;

    // 5. .mxl-файл рядом с фичей
    Если ЗначениеЗаполнено(ИмяФичи) Тогда
        // ... поиск файла ...
    КонецЕсли;

    Возврат Неопределено;
КонецФункции
```

**Лайфхак:** Chain of Responsibility — каждый уровень проверяет наличие ресурса, и если не находит — передаёт следующему. Позволяет переопределять макеты на любом уровне.

### 15.5 Что НЕ стоит копировать

| Паттерн | Где | Почему не нужно |
|---|---|---|
| `#Область include_core_...` | СБИС | Специфика системы сборки — для обычной обработки избыточна |
| `Вычислить()` / `Выполнить()` для вызова методов | Диадок | Теряется навигация по коду, нет статического анализа |
| 100+ переменных в одной форме | Vanessa | Вынужденная мера при 108 формах; для 10–30 достаточно нескольких форм-модулей |
| Синхронизация контекста через отдельный вызов | Диадок | Нужно только при вызове методов из другой независимой обработки |

### 15.6 Ключевые правила

1. **Форма = модуль.** Невидимая форма через `ПолучитьФорму()` — единственный способ организовать общий код в EPF.
2. **Контракт модуля:** `ИнициализацияФормы()` + `ОтключениеМодуля()` + экспортные методы.
3. **ObjectModule — для сервера.** Состояние, бизнес-логика, работа с базой данных.
4. **Переменные формы — для клиента.** Кэши, ссылки на модули, временные данные.
5. **Области для структуры.** `#Область` разделяет код по логическим блокам.
6. **UID для оповещений.** Всегда передавайте и проверяйте УникальныйИдентификатор.
7. **Lazy loading.** Загружайте модули при первом обращении, не при открытии формы.
8. **ФиксированнаяСтруктура для конфигов.** Защита от случайных изменений.
9. **Обёртки ЖР с префиксом.** `"МояОбработка." + ВидОперации` для фильтрации.
10. **Макеты в ПриСозданииНаСервере.** Загружайте все серверные данные один раз.
11. **DI вместо Service Locator.** Передавайте зависимости явно через `Инициализация(Владелец)`, а не через глобальный кеш. Явные зависимости = понятный контракт модуля и отсутствие скрытых связей.
