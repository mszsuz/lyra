# Lyra-Billing — биллинг и учёт запросов

Общайся с пользователем на русском языке.

## Что это

Подсистема учёта запросов и тарификации. Расширение конфигурации 1С. Каждый запрос к Claude уменьшает баланс клиента. Пока баланс > 0 — всё работает.

## Клиенты биллинга

- **Роутер** — проверяет баланс при авторизации сессии, списывает за запросы
- **Мобильное приложение** — отображает актуальный баланс пользователю, кнопка пополнения
- **Сайт** — личный кабинет, история запросов, пополнение

## Роль в архитектуре

Биллинг — WebSocket-клиент Centrifugo. Подписан на служебный канал `service:events`. Когда Роутер публикует `event_router_auth_completed` (с `session_id`, `user_id`, `channel`), Биллинг:

1. Проверяет баланс пользователя
2. Подключается к каналу сессии (WebSocket subscribe)
3. Публикует `event_billing_balance_checked` в `service:events`:
   `{type: "event_billing_balance_checked", session_id, status: "ok"|"insufficient_balance", balance: 1250.00}`
4. Роутер на основе этого ответа отправляет `auth_ack` в канал сессии
   - Таймаут ответа Биллинга при auth — 10 секунд. Если не ответил — Роутер отправляет `auth_ack` с `status: "service_unavailable"`
5. Биллинг отправляет `balance_update` в канал сессии при каждом изменении баланса
6. После каждого ответа Claude Биллинг списывает стоимость запроса и отправляет `balance_update` в канал сессии. Чат отображает баланс на форме
7. При пополнении до достаточного уровня — публикует `event_billing_balance_checked ok` → Роутер отправляет `auth_ack ok`

## Тарификация

| Модель | Input (1M токенов) | Output (1M токенов) |
|--------|-------------------|-------------------|
| Haiku 4.5 | $0.80 | $4 |
| Sonnet 4.5 | $3 | $15 |
| Opus 4.6 | $15 | $75 |

Примерная стоимость запроса:
- Простой (Haiku) — ~0.05 руб
- Сложный (Sonnet + 5 MCP) — ~7 руб
- Максимальный (Opus) — ~36 руб
