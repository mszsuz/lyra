# Остатки на расчётных счетах (счёт 51)

## Рабочий запрос — остатки по организациям

```bsl
ВЫБРАТЬ
	Остатки.Организация.Наименование КАК Организация,
	Остатки.Счет.Код КАК КодСчета,
	Остатки.СуммаОстатокДт КАК Остаток
ИЗ
	РегистрБухгалтерии.Хозрасчетный.Остатки КАК Остатки
ГДЕ
	Остатки.Счет.Код = "51"
	И Остатки.СуммаОстатокДт <> 0
УПОРЯДОЧИТЬ ПО
	Организация
```

## Важные особенности

1. **Виртуальная таблица `.Остатки` без параметров** — работает. С параметрами `(, Счет.Код = "51", , )` — возвращает пустой результат. Фильтрацию по счёту делать через `ГДЕ`.

2. **Субконто по банковским счетам** — при добавлении `Субконто1` в SELECT с фильтром по счёту 51, виртуальная таблица возвращает пустой результат. Даже с указанием ВидСубконто через `ЗНАЧЕНИЕ(ПланВидовХарактеристик.ВидыСубконтоХозрасчетные.БанковскиеСчета)`. Причина не установлена — возможно, особенность конфигурации или отсутствие аналитики по субконто на счёте 51 в данной базе.

3. **Основная таблица регистра** — поле `Счет` недоступно напрямую (ошибка "Поле не найдено"). Доступны `СчетДт` и `СчетКт`. Поле `Счет` доступно только в виртуальных таблицах (Остатки, Обороты и т.д.).

4. **Банковские счета организаций** можно получить отдельным запросом:
```bsl
ВЫБРАТЬ
	БС.Владелец.Наименование КАК Организация,
	БС.НомерСчета КАК НомерСчета,
	БС.Банк.Наименование КАК Банк
ИЗ
	Справочник.БанковскиеСчета КАК БС
ГДЕ
	НЕ БС.ПометкаУдаления
	И БС.Владелец ССЫЛКА Справочник.Организации
```

## Готовая функция в конфигурации

`Обработки.ОплатаПоставщикам.ОстаткиДенегНаРасчетныхСчетах(ДатаОстатков, Организация, БанковскийСчет)` — использует параметр `&СчетаССубконтоБС` из `УчетДенежныхСредствПовтИсп.ПолучитьСчетаССубконтоБанковскиеСчета()` и ВидСубконто `БанковскиеСчета`. Возвращает одно число (сумму).

## Смежные счета

- **50.01** — Касса организации
- **51** — Расчётные счета
- **52** — Валютные счета
- **57.03** — Продажи по платежным картам
