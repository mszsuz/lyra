# Правила запросов к виртуальной таблице Остатки

## Применимость

Эти правила действуют для ЛЮБЫХ запросов остатков:
- `РегистрБухгалтерии.Хозрасчетный.Остатки` (счёт 51, 50, 52, 62, 60, 41 — любой)
- `РегистрНакопления.*.Остатки`

## Правило 1: ВСЕГДА подставляй дату

Из документации 1С: **«Если параметр не указан, то будут получены текущие остатки (на максимальную дату движений регистра)»**.

Это НЕ текущая дата — если в базе есть документы из будущего, результат будет неверным.

**Действие:** при запросе «покажи остатки» молча подставляй завтрашнюю дату: `ДАТАВРЕМЯ(год, месяц, день+1)` от текущей даты. Если пользователь указал конкретную дату — используй её.

## Правило 2: ДАТАВРЕМЯ без времени = полночь

`ДАТАВРЕМЯ(2026, 3, 15)` = `15.03.2026 00:00:00`. Остатки считаются строго ДО этого момента. Движения в эту секунду и позже — не учтены.

Поэтому для остатков «на конец 14 марта» (включая все движения 14-го) → `ДАТАВРЕМЯ(2026, 3, 15)`.

## Правило 3: Дата через ДАТАВРЕМЯ(), не через параметр

Передача даты строкой через `&Параметр` не работает — несоответствие типов. Использовать `ДАТАВРЕМЯ()` прямо в тексте запроса.
