## 8. Система заявок на доработку
Когда пользователь просит написать код — Лира предлагает создать заявку для специалистов. Поток: пользователь соглашается → Sonnet внутренне анализирует задачу, формирует ТЗ → создаётся заявка → пользователь получает только название + ссылку (техническое описание не показывается). Требуется: отдельный сервис заявок (API), инструмент `create_ticket`, обновление промптов Haiku и Sonnet, новый тип сообщения Bridge → 1С со ссылкой. Также: userRole (user/developer) — выбор при регистрации в мобильном, влияет на стиль общения.

## 9. Права доступа / RLS
Как работать с правами пользователя? Варианты: работать от имени текущего пользователя (RLS автоматически), отдельный пользователь "Ассистент", от имени админа. Рекомендация: от имени текущего пользователя — RLS через `Запрос.Выполнить()` применяется автоматически.
