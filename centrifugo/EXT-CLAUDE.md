# ЕХТ_Центрифуга — расширение 1С для работы с Centrifugo

Общайся с пользователем на русском языке.

## Что это

Общее расширение конфигурации 1С, инкапсулирующее всё взаимодействие с Centrifugo. Другие серверные расширения (Роутер, Биллинг и др.) используют его как библиотеку — не работают с Centrifugo напрямую.

## Зачем

- Единая точка интеграции с Centrifugo — без дублирования кода в каждом расширении
- Изоляция протокола — если Centrifugo обновится или сменится транспорт, меняется только это расширение
- Переиспользование — любой серверный компонент Lyra подключает расширение и получает готовые методы

## Потребители

| Компонент | Что использует |
|-----------|---------------|
| **Роутер** | Server API (publish, subscribe), WebSocket-клиент (lobby, каналы сессий), JWT-генерация |
| **Биллинг** | WebSocket-клиент (service:events, каналы сессий), publish (balance_update, events) |
| **Другие** | Любой серверный компонент, которому нужен Centrifugo |

## Общие модули

### ЕХТ_Центрифуга (основной API)

Программный интерфейс для серверных компонентов.

#### Server API (HTTP)

Обёртки над HTTP Server API Centrifugo. Аутентификация через `X-API-Key`.

```
// Публикация сообщения в канал
ЕХТ_Центрифуга.Publish(Сервер, Канал, Данные)

// Подписать конкретного клиента на канал (для доставки hello_ack)
ЕХТ_Центрифуга.Subscribe(Сервер, Пользователь, КлиентИД, Канал)

// Отписать клиента от канала
ЕХТ_Центрифуга.Unsubscribe(Сервер, Пользователь, КлиентИД, Канал)

// Получить список присутствующих на канале
ЕХТ_Центрифуга.Presence(Сервер, Канал)

// Получить историю сообщений канала
ЕХТ_Центрифуга.History(Сервер, Канал)
```

Все методы — синхронные HTTP-вызовы. Формат ответа Centrifugo v6: POST на `/api/<метод>`, тело JSON, ответ JSON.

#### JWT

Генерация JWT-токенов, подписанных HMAC SHA-256 (`hmac_secret_key` из config.json Centrifugo).

```
// Сгенерировать JWT для подключения к Centrifugo
// Параметры: Сервер (ссылка), Sub (строка), Каналы (массив строк), ВремяЖизни (число секунд, 0 = без ограничения)
ЕХТ_Центрифуга.СоздатьJWT(Сервер, Sub, Каналы, ВремяЖизни)
```

JWT-структура:
```json
{
  "sub": "chat-<uuid>",
  "channels": ["session:<session_id>"],
  "exp": 1234567890
}
```

Реализация через платформенный объект `ТокенДоступа` (с 8.3.21):

```bsl
Токен = Новый ТокенДоступа();
Токен.КлючСопоставленияПользователя = Sub;  // claim "sub"
Токен.ВремяЖизни = ВремяЖизни;              // claim "exp" (секунды)
Токен.ПолезнаяНагрузка.Вставить("channels", Каналы);
Токен.Подписать(АлгоритмПодписиТокенаДоступа.HS256, СекретHMAC_Base64);
```

Ручная реализация HMAC/Base64URL не требуется.

### WebSocket-клиент (два способа)

Расширение предоставляет **два способа** подключения к Centrifugo по WebSocket — серверный (через метаданные) и универсальный (программный API).

#### Способ 1. Объект метаданных `WebSocketКлиент` (серверный)

Для серверных компонентов с постоянным соединением (Роутер, Биллинг). Платформа 8.3.27+.

**Доступность:** сервер, толстый клиент, внешнее соединение. **НЕ** доступен на тонком/веб-клиенте.

Преимущества:
- `ПодключатьАвтоматически = Истина` — платформа сама подключает при запуске и **переустанавливает соединение при разрыве**
- Собственный модуль с обработчиками событий — чистый серверный код
- `ПередПодключением` — позволяет менять URL/заголовки перед каждым подключением (и реконнектом)

Объект метаданных `ЕХТ_Центрифуга_Centrifugo` (WebSocketКлиент) в расширении:
- `URLСервера` — задаётся динамически в `ПередПодключением` из справочника
- `ПодключатьАвтоматически = Истина`
- Модуль с обработчиками:

```bsl
// Перед каждым подключением (и реконнектом) — подставить URL из справочника
Процедура ПередПодключением(Клиент)
    Сервер = ЕХТ_Центрифуга_Конфигурация.СерверПоУмолчанию();
    Клиент.URLСервера = ЕХТ_Центрифуга_Конфигурация.URLВебСокет(Сервер);
КонецПроцедуры

// Соединение установлено — отправить connect с JWT
Процедура ПриОткрытииСоединения(Соединение)
    Сервер = ЕХТ_Центрифуга_Конфигурация.СерверПоУмолчанию();
    JWT = ЕХТ_Центрифуга.СоздатьJWT(Сервер, "router", , 0);
    Соединение.ОтправитьСообщение("{""id"":1,""connect"":{""token"":""" + JWT + """}}");
КонецПроцедуры

// Получение сообщения — разбор протокола Centrifugo
Процедура ПриПолученииСообщения(Соединение, Сообщение)
    // Мульти-JSON: разбить по \n, развернуть push-конверт, dispatch
КонецПроцедуры

Процедура ПриОшибке(Соединение, КодОшибки, Описание)
    // Логирование
КонецПроцедуры

Процедура ПриЗакрытииСоединения(Соединение, КодЗакрытия)
    // Платформа сама переподключит (ПодключатьАвтоматически = Истина)
КонецПроцедуры
```

Потребители (Роутер, Биллинг) могут создавать **свои** объекты `WebSocketКлиент` в своих расширениях, используя общие модули ЕХТ_Центрифуга для протокола. Либо использовать предопределённый клиент из ЕХТ_Центрифуга.

#### Способ 2. Программный API `WebSocketКлиентСоединения` (универсальный)

Для клиентского кода (Чат EPF, мобильное) и случаев, когда нужно управлять жизненным циклом вручную.

**Доступность:** тонкий клиент, веб-клиент, сервер, толстый клиент, внешнее соединение.

Общий модуль `ЕХТ_Центрифуга_ВебСокетКлиент` (клиент + сервер):

```
// Открыть подключение к Centrifugo (асинхронно)
ЕХТ_Центрифуга_ВебСокетКлиент.Подключиться(Сервер, JWT, КлючСоединения)

// Подписаться на канал (через WebSocket-команду Centrifugo)
ЕХТ_Центрифуга_ВебСокетКлиент.Подписаться(КлючСоединения, Канал)

// Опубликовать сообщение (через WebSocket-команду Centrifugo)
ЕХТ_Центрифуга_ВебСокетКлиент.Опубликовать(КлючСоединения, Канал, Данные)

// Отключиться
ЕХТ_Центрифуга_ВебСокетКлиент.Отключиться(КлючСоединения)
```

Модель — **событийная**. `WebSocketКлиентСоединения.ОткрытьСоединение()` вызывает обработчики при получении данных. Обработчики задаются через `ОбработчикиWebSocketКлиентСоединения`, модуль — общий модуль или форма.

Готовый паттерн: `Chat/МодульBridge` (175 строк) — уже работает с WebSocket 8.3.27.

#### Общее для обоих способов (протокол Centrifugo)

- Протокол Centrifugo JSON поверх WebSocket: `{"id": N, "connect": {"token": "..."}}`, `{"id": N, "subscribe": {"channel": "..."}}`
- **Мульти-JSON** — Centrifugo может отправлять несколько JSON в одном WS-фрейме через `\n`. Клиент разбивает по `\n` и парсит каждую строку отдельно
- **Push-конверт** — входящие push обёрнуты: `{"push": {"channel": "...", "pub": {"data": {...}}}}`. Клиент разворачивает до `data`
- **Ping/pong** — Centrifugo шлёт `{}` (пустой JSON), клиент отвечает `{}`. Интервал настраивается в config.json
- **Автоинкремент id** — каждый запрос имеет уникальный `id`, ответы содержат тот же `id`

Логика разбора протокола (мульти-JSON, push-конверт, ping/pong) — общая, вынесена в служебные функции `ЕХТ_Центрифуга_ВебСокетКлиент`. Оба способа используют одни и те же функции разбора

### Справочник ЕХТ_Центрифуга (справочник)

Хранит настройки подключения к инстансам Centrifugo. Позволяет работать с несколькими серверами (dev, prod, отдельный для мобильных и т.д.).

| Реквизит | Тип | Описание |
|----------|-----|----------|
| Наименование | Строка | Название инстанса (например «Продакшн», «Разработка») |
| URLВебСокет | Строка | `ws://host:port/connection/websocket` |
| URLСерверAPI | Строка | `http://host:port/api` |
| КлючAPI | Строка | `X-API-Key` для Server API |
| СекретHMAC | Строка | `hmac_secret_key` для подписи JWT (HMAC SHA-256) |

Методы API принимают ссылку на сервер как первый параметр:

```
ЕХТ_Центрифуга.Publish(Сервер, Канал, Данные)
ЕХТ_Центрифуга.Subscribe(Сервер, Пользователь, КлиентИД, Канал)
ЕХТ_Центрифуга.СоздатьJWT(Сервер, Sub, Каналы, ВремяЖизни)
Клиент = ЕХТ_Центрифуга_ВебСокетКлиент.Подключиться(Сервер, JWT)
```

### ЕХТ_Центрифуга_Конфигурация (настройки)

Вспомогательный модуль для работы со справочником серверов.

```
// Получить сервер по умолчанию (если один инстанс)
Сервер = ЕХТ_Центрифуга_Конфигурация.СерверПоУмолчанию()

// Получить настройки сервера
ЕХТ_Центрифуга_Конфигурация.URLВебСокет(Сервер)
ЕХТ_Центрифуга_Конфигурация.URLСерверAPI(Сервер)
ЕХТ_Центрифуга_Конфигурация.КлючAPI(Сервер)
ЕХТ_Центрифуга_Конфигурация.СекретHMAC(Сервер)
```

## Что НЕ входит в расширение

- **Бизнес-логика** — сессии, авторизация, биллинг. Это ответственность потребителей
- **Конфигурация Centrifugo-сервера** — config.json, запуск, администрирование

## Зависимости

- Платформа 1С:Предприятие 8.3.27+ (WebSocket-клиент `НовыйОбъект("WebSocketКлиентСоединения")`)
- Centrifugo v6 (JSON-протокол поверх WebSocket, Server API v6)
