# ЕХТ_Центрифуга — расширение 1С для работы с Centrifugo

Общайся с пользователем на русском языке.

## Что это

Общее расширение конфигурации 1С, инкапсулирующее всё взаимодействие с Centrifugo. Другие серверные расширения (Роутер, Биллинг и др.) используют его как библиотеку — не работают с Centrifugo напрямую.

## Зачем

- Единая точка интеграции с Centrifugo — без дублирования кода в каждом расширении
- Изоляция протокола — если Centrifugo обновится или сменится транспорт, меняется только это расширение
- Переиспользование — любой серверный компонент Lyra подключает расширение и получает готовые методы

## Потребители

| Компонент | Что использует |
|-----------|---------------|
| **Роутер** | Server API (publish, subscribe), WebSocket-клиент (lobby, каналы сессий), JWT-генерация |
| **Биллинг** | WebSocket-клиент (service:events, каналы сессий), publish (balance_update, events) |
| **Другие** | Любой серверный компонент, которому нужен Centrifugo |

## Общие модули

### ЕХТ_Центрифуга (основной API)

Программный интерфейс для серверных компонентов.

#### Server API (HTTP)

Обёртки над HTTP Server API Centrifugo. Аутентификация через `X-API-Key`.

```
// Публикация сообщения в канал
ЕХТ_Центрифуга.Publish(Канал, Данные)

// Подписать конкретного клиента на канал (для доставки hello_ack)
ЕХТ_Центрифуга.Subscribe(Пользователь, КлиентИД, Канал)

// Отписать клиента от канала
ЕХТ_Центрифуга.Unsubscribe(Пользователь, КлиентИД, Канал)

// Получить список присутствующих на канале
ЕХТ_Центрифуга.Presence(Канал)

// Получить историю сообщений канала
ЕХТ_Центрифуга.History(Канал)
```

Все методы — синхронные HTTP-вызовы. Формат ответа Centrifugo v6: POST на `/api/<метод>`, тело JSON, ответ JSON.

#### JWT

Генерация JWT-токенов, подписанных HMAC SHA-256 (`hmac_secret_key` из config.json Centrifugo).

```
// Сгенерировать JWT для подключения к Centrifugo
// Параметры: Sub (строка), Каналы (массив строк), ВремяЖизни (число секунд, 0 = бессрочный)
ЕХТ_Центрифуга.СоздатьJWT(Sub, Каналы, ВремяЖизни)
```

JWT-структура:
```json
{
  "sub": "chat-<uuid>",
  "channels": ["session:<session_id>"],
  "exp": 1234567890
}
```

Подпись: HMAC SHA-256 (алгоритм HS256). Реализация в BSL: Base64URL-кодирование header + payload, подпись через платформенные средства или внешнюю компоненту.

### ЕХТ_Центрифуга_ВебСокетКлиент (WebSocket-клиент)

WebSocket-клиент для серверных компонентов. Позволяет подписываться на каналы и получать сообщения.

```
// Создать подключение к Centrifugo
// JWT — токен для аутентификации (серверный, с нужными правами)
Клиент = ЕХТ_Центрифуга_ВебСокетКлиент.Подключиться(URL, JWT)

// Подписаться на канал
ЕХТ_Центрифуга_ВебСокетКлиент.Подписаться(Клиент, Канал)

// Опубликовать сообщение
ЕХТ_Центрифуга_ВебСокетКлиент.Опубликовать(Клиент, Канал, Данные)

// Получить входящие сообщения (неблокирующий опрос)
Сообщения = ЕХТ_Центрифуга_ВебСокетКлиент.ПолучитьСообщения(Клиент)

// Отключиться
ЕХТ_Центрифуга_ВебСокетКлиент.Отключиться(Клиент)
```

Внутри:
- Протокол Centrifugo JSON поверх WebSocket: `{"id": N, "connect": {"token": "..."}}`, `{"id": N, "subscribe": {"channel": "..."}}`
- **Мульти-JSON** — Centrifugo может отправлять несколько JSON в одном WS-фрейме через `\n`. Клиент разбивает по `\n` и парсит каждую строку отдельно
- **Push-конверт** — входящие push обёрнуты: `{"push": {"channel": "...", "pub": {"data": {...}}}}`. Клиент разворачивает до `data`
- **Ping/pong** — Centrifugo шлёт `{}` (пустой JSON), клиент отвечает `{}`. Интервал настраивается в config.json
- **Автоинкремент id** — каждый запрос имеет уникальный `id`, ответы содержат тот же `id`

### ЕХТ_Центрифуга_Конфигурация (настройки)

```
// URL Centrifugo (ws:// для WebSocket, http:// для Server API)
ЕХТ_Центрифуга_Конфигурация.URLВебСокет()    // "ws://localhost:11000/connection/websocket"
ЕХТ_Центрифуга_Конфигурация.URLСерверAPI()   // "http://localhost:11000/api"
ЕХТ_Центрифуга_Конфигурация.КлючAPI()        // X-API-Key для Server API
ЕХТ_Центрифуга_Конфигурация.СекретHMAC()     // hmac_secret_key для подписи JWT
```

Хранение: константы или регистр сведений расширения. Настраиваются при установке.

## Что НЕ входит в расширение

- **Бизнес-логика** — сессии, авторизация, биллинг. Это ответственность потребителей
- **Клиентский мини-клиент BSL** (в Chat/EPF) — он работает на стороне клиента 1С, а не на сервере. Отдельная реализация в МодульBridge
- **Конфигурация Centrifugo-сервера** — config.json, запуск, администрирование

## Зависимости

- Платформа 1С:Предприятие 8.3.27+ (WebSocket-клиент `НовыйОбъект("WebSocketКлиентСоединения")`)
- Centrifugo v6 (JSON-протокол поверх WebSocket, Server API v6)
