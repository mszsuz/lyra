## Порядок разработки

### Этап 1. Справочник + Конфигурация

**Цель:** можно создать элемент справочника с настройками и прочитать их программно.

1. Добавить реквизиты в справочник `ЕХТ_Центрифуга`:
   - `URLВебСокет` (Строка, 250)
   - `URLСерверAPI` (Строка, 250)
   - `КлючAPI` (Строка, 250)
   - `СекретHMAC` (Строка, 250)

2. Обновить форму элемента — вывести реквизиты на форму

3. Создать общий модуль `ЕХТ_Центрифуга_Конфигурация`:
   - `СерверПоУмолчанию()` — первый (единственный) элемент справочника
   - `URLВебСокет(Сервер)`, `URLСерверAPI(Сервер)`, `КлючAPI(Сервер)`, `СекретHMAC(Сервер)` — чтение реквизитов через `ЕХТ_Ссылка.ЗначенияРеквизитов`

**Проверка:** создать элемент справочника, заполнить, прочитать программно.

### Этап 2. Server API (HTTP)

**Цель:** из 1С можно вызвать Centrifugo Server API (publish, subscribe и т.д.).

1. Служебная функция `ВызовServerAPI(Сервер, Метод, ТелоЗапроса)`:
   - HTTP POST на `URLСерверAPI(Сервер)/метод`
   - Заголовок `X-API-Key: КлючAPI(Сервер)`
   - Тело: JSON
   - Возврат: структура ответа

2. Экспортные функции в `ЕХТ_Центрифуга`:
   - `Publish(Сервер, Канал, Данные)` — POST `/api/publish`
   - `Subscribe(Сервер, Пользователь, КлиентИД, Канал)` — POST `/api/subscribe`
   - `Unsubscribe(Сервер, Пользователь, КлиентИД, Канал)` — POST `/api/unsubscribe`
   - `Presence(Сервер, Канал)` — POST `/api/presence`
   - `History(Сервер, Канал)` — POST `/api/history`

**Проверка:** запустить Centrifugo, вызвать `ЕХТ_Центрифуга.Publish(Сервер, "test:ch", Новый Структура("msg", "hello"))` — увидеть в Admin UI.

### Этап 3. JWT

**Цель:** генерировать JWT-токены для клиентов Centrifugo.

Используется платформенный объект `ТокенДоступа` (с 8.3.21) — ручная реализация HMAC/Base64URL не нужна.

Экспортная функция `СоздатьJWT(Сервер, Sub, Каналы, ВремяЖизни)`:

```bsl
Токен = Новый ТокенДоступа();
Токен.КлючСопоставленияПользователя = Sub;  // claim "sub"
Токен.ВремяЖизни = ВремяЖизни;              // claim "exp"
Токен.ПолезнаяНагрузка.Вставить("channels", Каналы);

СекретBase64 = СекретHMAC(Сервер);  // из справочника, в формате Base64
Токен.Подписать(АлгоритмПодписиТокенаДоступа.HS256, СекретBase64);

Возврат Строка(Токен);  // готовый JWT-строка
```

**Нюанс:** `СекретHMAC` в справочнике должен храниться в формате Base64 (метод `Подписать` ожидает Base64-строку).

**Проверка:** сгенерировать JWT, подключиться к Centrifugo с ним через Node.js тест.

### Этап 4. WebSocket-клиент (два способа)

**Цель:** и серверные компоненты (Роутер, Биллинг), и клиентский код (Чат EPF) могут подключиться к Centrifugo по WebSocket.

Платформа 8.3.27 предоставляет **два механизма** WebSocket-клиента:

#### 4а. Объект метаданных `WebSocketКлиент` (серверный, с автоподключением)

Доступен: сервер, толстый клиент, внешнее соединение. **НЕ** доступен на тонком/веб-клиенте.

1. Создать объект метаданных `ЕХТ_Центрифуга_Centrifugo` (тип `WebSocketClient`) в расширении:
   - `Predefined = true`, `AutoConnect = false` (подключение по запросу потребителя, не при старте)
   - `ServerURL` — заглушка, реальный URL подставляется в `ПередПодключением`
   - `Timeout = 30`

2. Модуль WebSocket-клиента (5 обработчиков):

   ```bsl
   Процедура ПередПодключением(Клиент)
       // Подставить URL из справочника (вызывается и при реконнекте)
       Сервер = ЕХТ_Центрифуга_Конфигурация.СерверПоУмолчанию();
       Клиент.URLСервера = ЕХТ_Центрифуга_Конфигурация.URLВебСокет(Сервер);
   КонецПроцедуры

   Процедура ПриОткрытииСоединения(Соединение)
       // Отправить connect с JWT (протокол Centrifugo)
   КонецПроцедуры

   Процедура ПриПолученииСообщения(Соединение, Сообщение)
       // Разбор: мульти-JSON, push-конверт, ping/pong
   КонецПроцедуры

   Процедура ПриОшибке(Соединение, КодОшибки, Описание)
   КонецПроцедуры

   Процедура ПриЗакрытииСоединения(Соединение, КодЗакрытия)
   КонецПроцедуры
   ```

3. Доступ к клиенту (паттерн из примеров dsdred/WebSocketIn1C):

   ```bsl
   // Найти предопределённый клиент
   МетаКлиент = Метаданные.WebSocketКлиенты.Найти("ЕХТ_Центрифуга_Centrifugo");
   ВебсокетКлиент = WebSocketКлиенты.НайтиПредопределенный(МетаКлиент);

   // Если ещё не создан — создать
   Если ВебсокетКлиент = Неопределено Тогда
       ВебсокетКлиент = WebSocketКлиенты.СоздатьКлиента(МетаКлиент, НовыйКлюч);
       ВебсокетКлиент.Записать();
   КонецЕсли;

   // Подключить (вызовет ПередПодключением → ПриОткрытииСоединения)
   ВебсокетКлиент.Подключить();

   // Для отправки:
   Соединение = ВебсокетКлиент.ПолучитьТекущееСоединение();
   Соединение.ОтправитьСообщение(ТекстJSON);

   // Проверка состояния:
   Состояние = ВебсокетКлиент.ПолучитьСостояниеСоединения();
   // СостояниеWebSocketСоединения.Открыто / Закрыто
   ```

**Ключевое преимущество:** при `ПодключатьАвтоматически = Истина` платформа сама реконнектит при разрыве. Но для Centrifugo стартуем вручную (`AutoConnect = false`), чтобы потребитель решал когда подключаться.

#### 4б. Программный API `WebSocketКлиентСоединения` (универсальный)

Доступен: тонкий клиент, веб-клиент, сервер, толстый клиент, внешнее соединение.

1. Создать общий модуль `ЕХТ_Центрифуга_ВебСокетКлиент` (флаги: Клиент + Сервер)

2. `Подключиться(Сервер, JWT, КлючСоединения)`:
   - `WebSocketКлиентСоединения.ОткрытьСоединение(Ключ, URL, Обработчики)`
   - Обработчики через `ОбработчикиWebSocketКлиентСоединения` — модуль = `ЕХТ_Центрифуга_ВебСокетКлиент`
   - В `ОбработчикОткрытияСоединения` — отправить connect с JWT

3. `Подписаться(КлючСоединения, Канал)`, `Опубликовать(КлючСоединения, Канал, Данные)`, `Отключиться(КлючСоединения)` — обёртки над `Соединение.ОтправитьСообщение()`

4. Обработчики — экспортные процедуры в `ЕХТ_Центрифуга_ВебСокетКлиент`

#### Общая логика протокола

Функции разбора протокола Centrifugo вынесены в служебную область `ЕХТ_Центрифуга_ВебСокетКлиент` — используются обоими способами:

- `РазобратьВходящееСообщение(Текст)` — split по `\n`, parse JSON, развернуть push-конверт
- `СформироватьКоманду(Метод, Параметры, ИД)` — JSON-команда Centrifugo (connect, subscribe, publish)
- `ОбработатьPing(Соединение, Сообщение)` — ответить `{}` на пустой JSON

**Проверка:** Роутер подключается к lobby через метаданные (автоподключение), Чат — через программный API. Оба получают hello от Node.js-клиента.

### Этап 5. Форма настроек + UX

1. Обновить форму «Настройки» обработки:
   - Кнопка «Проверить подключение» — Server API ping или publish тестового сообщения

2. Форма элемента справочника:

## Граф зависимостей

```
Справочник ЕХТ_Центрифуга (реквизиты)
    │
    ▼
ЕХТ_Центрифуга_Конфигурация (чтение настроек)
    │
    ├──────────────────┬──────────────────────────────┐
    ▼                  ▼                              ▼
ЕХТ_Центрифуга     ЕХТ_Центрифуга_ВебСокетКлиент  WebSocketКлиент (метаданные)
(Server API + JWT)  (программный API, клиент+сервер) (серверный, автоподключение)
                    │                              │
                    └──────────┬───────────────────┘
                               ▼
                    Общая логика протокола Centrifugo
                    (разбор мульти-JSON, push-конверт, ping/pong)
```

Этапы 1 → 2 → 3 строго последовательные (каждый зависит от предыдущего).
Этап 4 (оба способа WS) зависит от Этапа 1 (Конфигурация) + Этапа 3 (JWT), можно параллельно с 2.
Этап 5 — после всех остальных.

## Решённые вопросы

1. **JWT** — платформенный `ТокенДоступа` (с 8.3.21), `АлгоритмПодписиТокенаДоступа.HS256`. Ручной HMAC/Base64URL не нужен.
2. **WebSocket (сервер)** — объект метаданных `WebSocketКлиент` (8.3.27). `ПодключатьАвтоматически` = reconnect из коробки. Модуль с 5 обработчиками. `ПередПодключением` — динамический URL/JWT.
3. **WebSocket (клиент + сервер)** — программный `WebSocketКлиентСоединения.ОткрытьСоединение()` (8.3.27), асинхронный с обработчиками событий. Готовый паттерн в `Chat/МодульBridge` (175 строк).
4. **Справочник** — в скелете назван `ЕХТ_Центрифуга` (не `ЕХТ_Центрифуга_Серверы`). Спецификация EXT-CLAUDE.md уже обновлена.
5. **Два способа WS** — оба нужны: метаданные для серверных компонентов (Роутер, Биллинг), программный API для клиентского кода (Чат EPF). Логика протокола Centrifugo — общая.
