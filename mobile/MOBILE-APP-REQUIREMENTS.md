# Мобильное приложение Lyra — требования

## Назначение

Приложение-компаньон для Lyra. Не дублирует чат — работает как устройство ввода для Чата (1С) и личный кабинет пользователя. Единственный способ авторизации сессии 1С (QR-код). По user_id идёт биллинг.

## Платформа

**Flutter** (iOS + Android, единая кодовая база).

Выбор обусловлен:
- Кроссплатформенность (оба магазина из одного кода)
- Зрелые пакеты: `speech_to_text`, `mobile_scanner`, `camera`, WebSocket
- Компиляция в нативный код — важно для realtime-задач (STT, WebSocket)

## Регистрация пользователя

При первом запуске:

1. Пользователь вводит номер телефона
2. Получает SMS с кодом подтверждения
3. Вводит код — номер подтверждён
4. Создаётся `user_id` в MDM, привязывается к номеру телефона
5. `user_id` сохраняется на устройстве в secure storage (Keychain / Keystore)

Восстановление на новом устройстве: ввод телефона → SMS-подтверждение → MDM возвращает существующий `user_id`.

## Авторизация сессии 1С (QR-флоу)

Мобильное приложение — **единственный** способ авторизации. Альтернатив (логин/пароль) нет.

### Что содержит QR-код

QR-код = **JWT-токен для Centrifugo** (подписан Роутером). Содержит:
- `sub` — уникальный идентификатор для мобильного соединения (`mobile-<session_uuid>`)
- `exp` — срок жизни 5 минут
- Доступ к персональному каналу сессии

По истечении 5 минут Чат автоматически запрашивает новый QR (повторный hello).

### Флоу

```
1. Чат (1С) → Centrifugo (общий JWT) → lobby → hello (данные о базе)
2. Роутер создаёт сессию, генерирует 2 JWT:
   - chat_jwt (sub=chat-<uuid>) — для Чата
   - mobile_jwt (sub=mobile-<uuid>) — для мобильного
   Оба дают доступ к одному каналу сессии.
3. Роутер → hello_ack: {session_id, chat_jwt, mobile_jwt}
4. Чат переподключается к Centrifugo с chat_jwt
5. Чат отображает QR-код = mobile_jwt
6. Пользователь сканирует QR мобильным приложением
7. Мобильное подключается к Centrifugo с mobile_jwt → попадает на канал сессии
8. Мобильное отправляет {user_id} на канале
9. Роутер связывает пользователя с сессией и базой → авторизация завершена
```

Мобильное **никогда не подключается к lobby** — сразу попадает на нужный канал через JWT из QR.

## Функции

### MVP

1. **Регистрация** — номер телефона + SMS-подтверждение
2. **Сканер QR** — сканирование QR-кода с формы Чата (1С)
3. **Баланс** — текущий остаток пользователя от биллинга
4. **Статус подключения** — какая база подключена / не подключена
5. **Список баз** — история подключений пользователя (базы, к которым подключался)

### Будущие

6. **Микрофон** — голосовой ввод: речь → текст → канал → Роутер → Claude (Чат видит на канале)
7. **Камера** — фото → канал → Роутер → Claude (Чат видит на канале и отображает)
8. **Статистика** — расходы по сессиям, архив чата
9. **Пополнение баланса** — внешний эквайринг (детали TBD)
10. **Push-уведомления** — баланс заканчивается

## Несколько сессий

Один пользователь может работать с **несколькими базами 1С одновременно**. Мобильное приложение хранит список баз с историей подключений.

## Медиа-поток (голос, фото)

```
Мобильное → публикует медиа в канал сессии
    → Роутер подхватывает → передаёт Claude
    → Чат видит (подписан на тот же канал) → отображает в чате
```

Все подписчики канала видят все сообщения — дополнительная маршрутизация не нужна.

## Транспорт

Подключается к **Centrifugo** по WebSocket с персональным JWT (из QR-кода). Канал сессии определяется JWT (поле `sub`).

## Оплата

Внешний эквайринг (не встроенные покупки Apple/Google). Детали — TBD.
