# Lyra — монорепо

Общайся с пользователем на русском языке.

## Что это

Lyra — ИИ-ассистент для 1С:Предприятие на базе Claude. Пользователи задают вопросы через чат прямо в интерфейсе 1С, Claude отвечает в реальном времени с доступом к данным базы.

Часть экосистемы [1ext.com](https://1ext.com) | [GitHub](https://github.com/mszsuz/lyra)

## Структура монорепо

```
Lyra/
├── Chat/          — Клиентское ПО: внешняя обработка 1С (чат-интерфейс)
├── Bridge/        — Серверное ПО: Node.js мост (прототип, заменяется Роутером)
├── Router/        — Серверное ПО: транспорт и маршрутизация (Centrifugo + расширение 1С)
├── mdm/           — Серверное ПО: общая НСИ (расширение 1С)
├── moderator/     — Серверное ПО: проверка сообщений (расширение 1С)
├── billing/       — Серверное ПО: биллинг и учёт запросов (расширение 1С)
├── site/          — Серверное ПО: сайт и личный кабинет (расширение 1С)
├── mobile/        — Клиентское ПО: мобильное приложение
├── Vega/          — Отдельный репо (github.com/mszsuz/Vega), в .gitignore
└── docs/          — Общая документация
    ├── prompts/       — системные промпты агентов
    ├── competitors/   — анализ конкурентов
    └── branding/      — материалы по бренду
```

## Архитектура

```
Мобильное приложение ──┐
Чат (1С) ──────────────┼──► Centrifugo ──► ЕХТ_Лира_Роутер (1С) ──► Модератор ──► Claude (Haiku/Sonnet/Opus)
Сайт ──────────────────┘        │                    │
                                │                    ├──► MDM (авторизация, НСИ)
                                │                    ├──► Биллинг (баланс, тарификация)
                                │                    └──► Vega (MCP, метаданные конфигураций)
                                │
                                └──► exec-команды ──► 1С клиента (через Centrifugo)
```

- **Роутер** — центральный хаб: Centrifugo (транспорт) + ЕХТ_Лира_Роутер (логика в 1С). Заменяет Bridge
- **Bridge** — прототип (Node.js), использовался для отладки. Роутер берёт на себя его функции
- **Vega** — парсер конфигураций 1С + MCP-сервер для AI-агентов (Rust, Neo4j)
- **MDM** — общая НСИ для всех компонентов (пользователи, организации, тарифы)
- **Модератор** — проверка входящих сообщений отдельной моделью, соблюдение легенды «Лира»
- **Биллинг** — учёт запросов, баланс клиентов, тарификация

## Правила работы

- Каждый подпроект имеет свой CLAUDE.md с деталями
- Спецификации протоколов — в docs/ или в CLAUDE.md подпроекта
- При кросс-проектных изменениях (протокол, API) обновляй все затронутые подпроекты в одном коммите
- Vega — отдельный git-репо, не коммитить в этот репо
