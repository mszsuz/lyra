# Lyra — монорепо

Общайся с пользователем на русском языке.

## Что это

Lyra — ИИ-ассистент для 1С:Предприятие на базе Claude. Пользователи задают вопросы через чат прямо в интерфейсе 1С, Claude отвечает в реальном времени с доступом к данным базы.

Часть экосистемы [1ext.com](https://1ext.com) | [GitHub](https://github.com/mszsuz/lyra)

## Структура монорепо

```
Lyra/
├── TEST-LYRA/     — Рабочий каталог (код + данные + centrifugo), запуск отсюда
│   ├── centrifugo/    — бинарник + config.json (порт 11911, dev-секреты)
│   ├── router/        — код роутера + профили + config.json
│   ├── users/         — данные пользователей (cwd Claude CLI)
│   ├── var/           — логи, lobby-сессии
│   └── memory/        — общая память по конфигурациям
├── Chat/          — Исходники: внешняя обработка 1С (чат-интерфейс)
├── mobile/        — Исходники: мобильное приложение (Flutter, iOS + Android)
├── docs/          — Документация, бэклог, аудит безопасности, деплой
├── Vega/          — Отдельный репо (github.com/mszsuz/Vega), в .gitignore
├── _LEGASY/       — Архив старого кода (Bridge, DB, mdm, moderator)
└── C:\LYRA\       — Prod-копия TEST-LYRA (другой config.json, другие секреты)
```

## Рабочая среда

Код разрабатывается и тестируется в `TEST-LYRA/`. Перед продом — копируется в `C:\LYRA\`.

- **TEST-LYRA/router/** — рабочий код роутера. Все .mjs файлы идентичны `C:\LYRA\router\`
- **config.json** — единственное различие: порт, модель, профиль, dataDir
- **system-prompt.md** — может отличаться (founder vs user)
- Данные пользователей хранятся в `TEST-LYRA/users/`, логи в `TEST-LYRA/var/`
- Centrifugo запускается из `TEST-LYRA/centrifugo/`

Порт: **11911** (и dev, и prod). Одновременно запускается только одна Лира.

## Архитектура

```
Мобильное приложение ──┐
Чат (1С) ──────────────┼──► Centrifugo ──► Node.js Router ──► Claude CLI
Сайт ──────────────────┘     :11911          │                   ↕ MCP
                                             ├──► Vega (метаданные конфигураций)
                                             ├──► mcp-1c-docs (документация)
                                             └──► exec-команды ──► 1С клиента
```

- **Router** — Node.js, центральный хаб: Claude CLI (child process), MCP, авторизация
- **Vega** — парсер конфигураций 1С + MCP-сервер (Rust, Neo4j)
- **Centrifugo** — WebSocket-транспорт реального времени (v6)

## Правила работы

- Документация — в `docs/` (бэклог, архитектура, деплой, аудит)
- При кросс-проектных изменениях (протокол, API) обновляй все затронутые компоненты
- Vega — отдельный git-репо, не коммитить в этот репо
- TEST-LYRA/ и _LEGASY/ — в .gitignore
