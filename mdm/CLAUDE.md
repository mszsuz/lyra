# Lyra-MDM — общая НСИ

Общайся с пользователем на русском языке.

## Что это

Подсистема управления нормативно-справочной информацией (Master Data Management). Расширение конфигурации 1С. Единый источник НСИ для всех компонентов комплекса Lyra: пользователи, организации, тарифы, подписки, лицензии и другие общие справочники.

## Статус

В планах.

## Роль в архитектуре

MDM — фундамент для остальных подсистем. Все компоненты ссылаются на общую НСИ вместо дублирования данных.

```
MDM (общая НСИ)
 ├── Роутер     — пользователи, авторизация, идентификация баз
 ├── Биллинг    — тарифы, подписки, баланс
 ├── Сайт       — организации, личный кабинет
 └── Модератор  — настройки модерации
```

## Идентификация базы

При подключении Чат (1С) отправляет `hello` с данными о базе. Роутер регистрирует базу в MDM:

```
config_name + connection_string + computer → хеш → идентификатор базы в НСИ
```

| Поле | Пример | Описание |
|------|--------|----------|
| `config_name` | `БухгалтерияПредприятия` | Метаданные.Имя |
| `config_version` | `3.0.191.41` | Метаданные.Версия |
| `config_id` | `DEADBEEF-0000-0000-0000-00000000F00D` | Идентификатор конфигурации (стабильный UUID, не зависит от версии) |
| `computer` | `BUHPC-01` | ИмяКомпьютера() |
| `connection_string` | `Srvr="srv1c";Ref="buh_prod";` | СтрокаСоединенияИнформационнойБазы() |

Это позволяет:

- Различать одну и ту же конфигурацию на разных серверах
- Различать тестовую и рабочую базу на одном сервере
- Привязывать базу к тарифу, балансу, истории запросов
- Выбирать правильный Vega-инстанс (по конфигурации + версии)

## Справочники

MDM содержит 2 основных справочника:

- **Пользователи** — привязан к номеру телефона. Создаётся при SMS-регистрации через `mobile:lobby`
- **Базы** (подчинённый Пользователям) — создаётся при авторизации (шаг 7, auth). Привязка к пользователю

Дополнительно хранится `device_id` — для аудита.

## Пользователи

Пользователь регистрируется через мобильное приложение (канал `mobile:lobby`):

1. Вводит номер телефона
2. Получает SMS с кодом подтверждения
3. Подтверждает код → создаётся запись в справочнике Пользователи (`user_id`)

MDM хранит: `user_id`, номер телефона, `device_id`.

## Связь пользователя и базы

При QR-авторизации (шаг 7, auth) Роутер проверяет пару `user_id + device_id` в MDM. При ошибке — `auth_ack` с `status: "auth_failed"` (причина не раскрывается клиенту). При успехе — создаёт запись в подчинённом справочнике Базы, связывает пользователя с базой (из hello). MDM хранит эту связь.
