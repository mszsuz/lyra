# Коды ошибок Lyra

Ошибки отображаются в чате в формате: `Ошибка: описание (код XX). Действие.`

## Коды

| Код | Сообщение | Причина | Что делать |
|-----|-----------|---------|------------|
| 01 | Сервер не ответил вовремя | API-провайдер (OpenRouter, Anthropic) не ответил в течение `chunkTimeout` (по умолчанию 60 сек). Роутер сделал повторную попытку — тоже без ответа. | Повторить вопрос. Если повторяется — проверить статус провайдера (status.openrouter.ai). Проверить лог: `Adapter timeout [chunk/connect]` |
| 02 | Сервис временно недоступен | API-провайдер вернул ошибку (HTTP 4xx/5xx): rate limit, перегрузка, невалидный запрос. | Повторить через минуту. Если 429 (rate limit) — подождать. Если 401 — проверить API-ключ в profile.json. Лог: `Adapter error: API error {status}` |
| 03 | Превышен лимит обращений к данным | Модель сделала 10 tool calls подряд и не дала финальный ответ. Роутер отправил запрос без tools — модель всё равно попыталась вызвать инструмент. | Упростить вопрос, спросить конкретнее. Лог: `Model returned tool_use after tool limit` |
| 04 | Непредвиденная ситуация | Необработанное исключение в `runAdapterChatManaged()`. | Повторить вопрос. Если повторяется — смотреть stack trace в логе: `Adapter error: {message} {stack}` |
| 05 | Сервер не ответил вовремя | То же что код 01, но для passthrough-адаптеров (claude-cli, codex-cli). Retry не выполняется. | Повторить вопрос. Лог: `Passthrough adapter timeout` |
| 06 | Непредвиденная ситуация | Необработанное исключение в `runAdapterChatPassthrough()`. | Повторить вопрос. Лог: `Adapter error: {message} {stack}` |

## Где искать в логе

Лог роутера: `TEST-LYRA/var/router.log` (или `C:\LYRA\var\router.log` в production).

Каждая ошибка логируется **до** отправки сообщения пользователю:
- `[WARN]` — retry (код 01/05, первая попытка)
- `[ERROR]` — финальная ошибка (все коды)

Пример:
```
[WARN]  Adapter timeout [chunk] attempt 1/2, session abc123
[ERROR] Adapter timeout [chunk] after 2 attempts, session abc123
→ пользователь видит: "Ошибка: сервер не ответил вовремя (код 01). Попробуйте повторить."
```

## Настройки

`config.json` → `adapterTimeout`:
- `chunkTimeout` (мс) — макс. тишина между SSE-чанками. По умолчанию 60000
- `connectTimeout` (мс) — таймаут на соединение с API. По умолчанию 15000
- `maxRetries` — количество повторных попыток. По умолчанию 1
