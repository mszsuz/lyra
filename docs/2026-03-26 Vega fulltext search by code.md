# Vega: fulltext поиск по телам процедур

## Статус: СДЕЛАНО ✅ (2026-03-26)

Реализовано и протестировано. `find_routines_by_code` теперь поддерживает `search_mode: "fulltext"`.

## Результаты тестирования

### Запрос 1: `"Себестоимость"` по владельцу
```json
{"op": "find_routines_by_code", "text": "Себестоимость", "search_mode": "fulltext", "owner": "Документы.ВозвратТоваровОтПокупателя"}
```
→ **8 результатов**, включая `ПередЗаписью` (ранее vector search не находил)

### Запрос 2: `"УказанаПартия"` — точное вхождение переменной
```json
{"op": "find_routines_by_code", "text": "УказанаПартия", "search_mode": "fulltext", "owner": "Документы.ВозвратТоваровОтПокупателя"}
```
→ **3 результата**: `ПередЗаписью`, `ПодготовитьФормуНаСервере`, `УстановитьУсловноеОформление`

### Запрос 3: `"МассивНепроверяемыхРеквизитов"` — поиск валидации
```json
{"op": "find_routines_by_code", "text": "МассивНепроверяемыхРеквизитов", "search_mode": "fulltext", "owner": "Документы.ВозвратТоваровОтПокупателя"}
```
→ **1 результат**: `ОбработкаПроверкиЗаполнения`

## Что было раньше

`find_routines_by_code` поддерживал только vector (семантический) поиск. Запрос `"УказанаПартия Себестоимость"` через vector → **ничего не найдено**. Ключевая находка (очистка Себестоимости в `ПередЗаписью`) была получена только через grep по исходникам на диске, которого у Лиры нет.
