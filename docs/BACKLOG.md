## Загрузчик и подключение

- [ ] **Конфигурация подключения через GitHub** — JSON-файл в публичном репо (`github.com/mszsuz/lyra/config.json`) содержит адрес сервера, версию EPF и URL для скачивания. EPF и мобильное при запуске делают GET на `raw.githubusercontent.com/.../config.json`, получают актуальный `ws_url`. Менять сервер — один коммит в GitHub, без пересборки клиентов. Формат:
  ```json
  {"ws_url": "wss://lyra.1ext.com/connection/websocket", "epf_version": "26.03.22.2", "epf_url": "https://..."}
  ```
  Кэш в `ХранилищеОбщихНастроек` — если GitHub недоступен, подключаться по сохранённому адресу. GitHub CDN кэширует ~5 мин — допустимо для смены сервера. Когда появится свой домен — endpoint переедет на `https://lyra.1ext.com/config.json`, в GitHub — redirect
- [ ] **Автообновление обработки** — Загрузчик сравнивает `epf_version` из config.json с `Комментарий` метаданных. Если новее — предлагает скачать по `epf_url`, сохранить, переоткрыть

## Пошаговые HTML-инструкции (туториалы)

Концепция: Лира генерирует HTML-файл с интерактивной пошаговой инструкцией, визуально имитирующей интерфейс 1С. Пользователь спрашивает «покажи как сделать X» — получает туториал, где на каждом шаге видит макет экрана 1С с подсветкой элемента, на который нужно нажать, и пояснением справа.

**Прототип:** `docs/demo-tutorial-prototype.html` — демо «От поступления до реализации кабеля» (УТ 11, 12 шагов)

**Конкурентный контекст:** [coderstar.ru/kurs-ut11](https://www.coderstar.ru/kurs-ut11/) — традиционный формат (реальные скриншоты + красные/жёлтые выноски). Наш подход — интерактивный HTML с навигацией, пошаговость, без скриншотов — выигрывает по интерактивности и автоматизируемости

### Архитектура генерации (Вариант 2 — статический шаблон + JSON)

Три подхода к генерации:
1. **Полная генерация HTML** — модель генерирует весь файл (~5000 токенов). Максимальная гибкость, дорого
2. **✅ Статический шаблон + JSON шагов** — шаблон с компонентами уже готов, модель генерирует только массив шагов (~1000 токенов). Оптимальный баланс: в 4-5 раз дешевле варианта 1, визуал стабильный
3. **Пребилд** — заранее готовые туториалы для типовых сценариев. Нулевая стоимость, но не масштабируется

Выбран **Вариант 2**: один HTML-шаблон (CSS + JS + компоненты экранов) + JSON-массив шагов от модели. Шаблон одинаковый для всех конфигураций, меняется только цветовая схема и данные шагов.

### Источники данных (дополняют друг друга)

| Источник | Что даёт | Пример |
|----------|---------|--------|
| **Vega MCP** | Навигационные пути, структура форм, реквизиты, типы | «Закупки → Документы закупки (все)», поля формы ПриобретениеТоваровУслуг |
| **Документация MCP** (`1c-config-*`) | Бизнес-логика, порядок действий, доменная специфика | Учёт кабеля: серии номенклатуры для бухт, метраж, цены за метр, политика учёта серий |
| **HTML-шаблон** | Визуальное отображение, интерактивность | Панель навигации, формы, подсветка, навигация по шагам |

Vega → точные пути и структура. Документация → бизнес-смысл и специфика домена. Оба нужны.

### Формат JSON-шага (генерируется моделью)

```json
{
  "hintTitle": "Раздел «Закупки»",
  "hint": "<p>Описание</p><div class='hint-action'>Действие</div>",
  "activeNav": "purchases",
  "hlNav": "purchases",
  "screen": "purchasesSection",
  "tabs": ["Начальная страница"],
  "activeTab": 0
}
```

Типы экранов: `home`, `*Section` (страница раздела), `*List` (журнал), `*Form` (форма документа/справочника), `done` (итог). Подсветка: `hlNav`, `hlLink`, `hlCmd`, `hlField`, `hlRow`, `hlTabCmd`.
