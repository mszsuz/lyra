## UI рабочая область (workspace)

Концепция: чат справа, рабочая область слева. Рабочая область — **программно генерируемые страницы** формы обработки с динамически добавляемым содержимым (реквизиты, элементы формы, таблицы). Лира через инструменты создаёт страницы и наполняет их данными. Внизу — вкладки страниц.

- [ ] **Архитектура рабочей области** — форма обработки с программно добавляемыми страницами (ДобавитьСтраницу). Каждый вызов ui_* создаёт новую страницу или обновляет существующую. Страницы содержат динамически добавленные элементы формы (ТабличныйДокумент, ПолеHTMLДокумента, ТаблицаФормы и т.д.)
- [ ] **Инструменты отображения (в рабочей области):**
  - `ui_table` — страница с ТабличнымДокументом (данные, форматирование, печать, экспорт в Excel)
  - `ui_chart` — страница с диаграммой (встроенные диаграммы 1С в ТабличномДокументе)
- [ ] **Инструменты навигации:**
  - `ui_open_list` — страница с ДинамическимСписком с заданными отборами. Самый частый запрос — «покажи документы за период по контрагенту»
  - `ui_open_form` — открыть форму объекта (перекрывается с `v8_navlink`, но может задавать параметры формы)
  - `ui_open_report` — страница с результатом отчёта (предзаполненные настройки)

## Клиентские инструменты (интерфейс пользователя)

Сейчас все инструменты серверные (`v8_*` → `lyra_*`, `&НаСервере`). Клиентские инструменты (`lyra_ui_*`) дают Лире доступ к интерфейсу — «глаза и руки» в окнах 1С. Выполняются на клиенте (`&НаКлиенте`).

- [ ] **Диспетчер tool_call — разделение серверных/клиентских** — сейчас все tool_call идут в `МодульMCP.ВыполнитьИнструмент()` на сервере. Нужно разделить: `v8_*` → сервер, `ui_*` → клиент. Оба возвращают `tool_result` обратно в канал
- [ ] **Инструменты чтения интерфейса:**

  - `ui_get_field_value` — прочитать значение конкретного поля формы
  - `ui_get_active_window` — информация об активном окне (заголовок, тип формы, объект метаданных)
- [ ] **Инструменты управления интерфейсом:**
  - `ui_activate_window` — активировать окно по заголовку/индексу
  - `ui_open_list` — открыть ДинамическийСписок с отборами (также в рабочей области)
  - `ui_open_form` — открыть форму объекта
  - `ui_open_report` — открыть отчёт с настройками

## Оптимизация промпта

- [ ] **"Один поиск — одно действие"** — усилить правило: получил результат из Vega → сразу выполняй запрос, не делай повторных search_code для перепроверки

## Будущее развитие

- [ ] **Чат в мобильном приложении** — полноценный чат с Claude прямо в мобильном. Пользователь может отойти от компьютера, продолжить общение голосом через телефон. Чат в 1С остаётся открытым (позже — работа в фоне). Канал сессии единый — мобильное видит весь стриминг Claude и отображает в своём чате. Голосовой ввод через STT уже в плане, нужен ещё UI чата и отображение ответов Claude
- [ ] **Темы оформления (скины)** — переключаемые темы в мобильном приложении. 7 дизайн-концептов уже готовы (Glassmorphism, Minimal, Neobrutalism, Cyberpunk, Organic, Neumorphism, Flat). Реализация через кастомный ThemeData во Flutter. MVP — одна тема + dark/light, потом 3-4 темы на выбор в настройках
- [ ] **Геймификация (ачивки)** — система достижений для пользователей Лиры. Ref: [habr.com/mosigra/228593](https://habr.com/ru/companies/mosigra/articles/228593/). Идеи:
  - **Онбординг**: «Первый вопрос», «Первый exec-запрос», «Подключил мобильное» — обучают функциям через награды
  - **Вовлечение**: «10 сессий», «Подключил 3 базы», «Голосовой вопрос» — мотивируют использовать разные возможности
  - **Экспертиза**: «100 запросов к метаданным», «Нашёл баг через v8_query» — показывают глубину использования
  - **Социальная роль**: видимые значки в профиле — «старожил», «бета-тестер», «power user»
  - **Лояльность**: серии дней использования, бонусы за безупречные серии (как в Мосигре — серии безошибочных заказов)
  - Отображение: мобильное приложение (профиль) + сайт (личный кабинет). В чате 1С — не нужно (отвлекает)
  - Без манипулятивной монетизации — не привязывать ачивки к платным действиям
