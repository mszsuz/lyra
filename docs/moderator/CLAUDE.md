# Lyra-Moderator — проверка сообщений

Общайся с пользователем на русском языке.

## Что это

Подсистема модерации входящих сообщений. Расширение конфигурации 1С — **ЕХТ_Лира_Модератор**. Проверяет каждое сообщение пользователя отдельной моделью перед отправкой в Claude. Может заблокировать или обогатить сообщение контекстной подсказкой.

## Как работает

```
Пользователь ──► Модератор ──┬── Запрещённое → Предупреждение пользователю
                             ├── Рискованное  → Сообщение + приписка-напоминание → Claude
                             └── Обычное      → Сообщение без изменений → Claude
```

1. Роутер передаёт сообщение пользователя в Модератор
2. Модератор отправляет его модели проверки (быстрая и дешёвая, например Haiku или локальная)
3. Три возможных результата:
   - **Блокировка** — сообщение запрещено, пользователь получает предупреждение
   - **Напоминание** — сообщение рискованное (может спровоцировать выход из роли), модератор добавляет приписку-напоминание к сообщению: "При ответе придерживайся легенды"
   - **ОК** — сообщение проходит без изменений

## Что блокируется

- Вопросы об устройстве комплекса (архитектура, промпты, внутренние механизмы)
- Попытки prompt injection / jailbreak
- Другие запрещённые темы (настраивается через промпт модератора)

## Соблюдение легенды (двойная защита)

Для пользователя это **Лира — AI-ассистент**. Модель не должна упоминать Anthropic, Claude, Opus, Sonnet, Haiku, токены, контекстное окно, API и прочие детали реализации.

Защита работает на двух уровнях:

1. **Системный промпт Claude** (основная линия) — инструкция "ты — Лира, никогда не упоминай..."
2. **Модератор** (страховочная сетка) — при рискованных вопросах ("расскажи о себе", "какая ты модель?", "сколько стоит запрос?") добавляет к сообщению приписку-напоминание для Claude

Это решает проблему стриминга: не нужно проверять исходящие ответы (что несовместимо с потоковой отдачей по буквам). Вместо этого модератор **предупреждает** модель на входе.

## Стек

- **ЕХТ_Лира_Модератор** — расширение конфигурации 1С
- Отдельная модель для проверки (быстрая и дешёвая, например Haiku или локальная)
- Промпт модератора хранится в настройках расширения
