## Детали по каждой находке

### A-01. Биллинг: устаревший документ → удалён

`billing/BILLING-SERVICE-REQUIREMENTS.md` описывал биллинг как HTTP-сервис для Bridge. Актуальная архитектура (биллинг как WebSocket-клиент Centrifugo через service:events) описана в `billing/CLAUDE.md`, `docs/lyra-architecture.md` и `docs/INTERACTION-SCHEMA.md`. Файл удалён как устаревший и вводящий в заблуждение.

### A-02. JWT TTL → 1 год

Принято решение: оба персональных JWT (chat_jwt, mobile_jwt) со сроком жизни **1 год** (`exp` = now + 365 дней). Бессрочные токены — плохая практика, 5 минут — устаревшее значение из раннего черновика.

Обновлены файлы:
- `mobile/MOBILE-APP-REQUIREMENTS.md` — 5 мин → 1 год, убран абзац про автоматический повторный hello
- `mobile/CLAUDE.md` — бессрочный → 1 год
- `Router/CLAUDE.md` — бессрочный → 1 год (3 места)
- `docs/lyra-architecture.md` — бессрочный → 1 год (таблица + описание + флоу)
- `docs/INTERACTION-SCHEMA.md` — бессрочный → 1 год
- `Chat/CLAUDE.md` — бессрочный → 1 год (таблица + флоу)
- `centrifugo/CLAUDE.md` — бессрочный → 1 год (таблица)

### A-03. MDM: база создаётся при auth → исправлено ранее

База в MDM появляется на этапе `auth` (когда известен пользователь), а не при `hello`. Документы синхронизированы.

### A-04. Lobby wording → уточнено

Мобильное не подключается к `session:lobby` (Чат-lobby), но подключается к `mobile:lobby` для регистрации. Формулировка в `docs/INTERACTION-SCHEMA.md` уточнена.

### A-05. Naming drift → унифицировано

Каноническое именование зафиксировано и применено во всех документах:
- MCP-инструменты: `v8_query`, `v8_eval`, `v8_metadata`, `v8_exec` (не `1c_*`)
- Транспорт CLI: `stdio-bridge` (не «адаптер», не `adapter.exe`)
- Тип сообщения: `confirm` (не `confirm_sms`)

Обновлены: `docs/INTERACTION-SCHEMA.md`, `docs/lyra-architecture.md`, `Chat/CLAUDE.md`, `Router/CLAUDE.md`.

### A-06. auth_ack перегружен → оставлено как есть

Все статусы auth_ack (`ok`, `auth_failed`, `insufficient_balance`, `service_unavailable`) — результаты единого процесса авторизации. Пока биллинг не подтвердил баланс, авторизация не завершена. Введение отдельного `session_status` усложнит протокол без реальной пользы. `balance_update` уже существует для обновлений баланса после авторизации.

### A-07. Битые ссылки → исправлено

- `Chat/README.md:179` — `Конкуренты/Aether Lab/...` → `../docs/competitors/Aether Lab/...`
- `Chat/BACKLOG.md:38` — локальные пути `../../../WORKS/...` → помечены как «локальные проекты, не в репо»
- `Chat/BACKLOG.md:72,80` — `../../Конкуренты/...` → `../docs/competitors/...`

### A-08. Покрытие подсистем → частично закрыто

- Модератор перенесён из отдельной папки `moderator/` в `Router/CLAUDE.md` как модуль расширения ЕХТ_Лира_Роутер
- `site/CLAUDE.md` — остаётся минимальным, подсистема в ранней стадии
- `mdm/CLAUDE.md`, `billing/CLAUDE.md` — достаточны для текущего этапа проектирования

### A-09. Маркеры статуса → не нужно сейчас

Проект на ранней стадии — всё либо «target», либо «MVP в работе». Деление current/target/legacy имеет смысл при наличии production-системы с историей. Устаревшие документы удаляются (как A-01), а не маркируются.

### A-10. Документ по безопасности → не нужно сейчас

Безопасность описана по месту: JWT-схема в `Router/CLAUDE.md`, rate limiting в `mobile/CLAUDE.md`, device_id верификация в `docs/INTERACTION-SCHEMA.md`. Отдельный `security.md` на этапе проектирования создаст ещё один документ для синхронизации. Создавать при наличии production-кода.
