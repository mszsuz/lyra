# Lyra — схема взаимодействия компонентов

## Общая схема

```
                         Сервер Lyra
  База пользователя      ┌─────────────────────────────────────────────────────┐
  ┌──────────────┐        │                                                     │
  │ 1С           │        │  Centrifugo (:8000)                                 │
  │              │  WS    │  ┌─────────────────┐                                │
  │ Lyra-Chat.epf├───────►│  │ Каналы:         │                                │
  │              │◄───────┤  │  session:<id>    │                                │
  │  exec-       │        │  └────────┬────────┘                                │
  │  команды     │        │           │                                         │
  └──────────────┘        │           ▼                                         │
                          │  ЕХТ_Лира_Роутер (расширение 1С, серверная база)    │
                          │  ┌──────────────────────────────────────────────┐   │
                          │  │                                              │   │
                          │  │  1. Регистрация базы в MDM                   │   │
                          │  │  2. Авторизация                              │   │
                          │  │  3. Проверка баланса (Биллинг)               │   │
                          │  │  4. Модератор → проверка сообщения              │   │
                          │  │  5. Выбор Vega-инстанса по конфигурации      │   │
                          │  │  6. Запуск адаптера через ЕХТ_СтдИО          │   │
                          │  │                                              │   │
                          │  └──────────┬───────────────────────────────────┘   │
                          │             │                                       │
                          │             ▼                                       │
                          │  Адаптер (exe)                                      │
                          │  ┌──────────────────────┐    ┌───────────────────┐  │
                          │  │ Claude               │    │ MCP-серверы       │  │
                          │  │ (Haiku/Sonnet/Opus)  │◄──►│  Vega             │  │
                          │  │                      │    │  mcp-1c-docs      │  │
                          │  └──────────────────────┘    └───────────────────┘  │
                          └─────────────────────────────────────────────────────┘
```

## Подключение: hello-рукопожатие

### 1. Пользователь открывает Lyra-Chat.epf в своей базе 1С

Обработка собирает информацию о базе:

| Поле | Пример | Описание |
|------|--------|----------|
| `configuration` | `БухгалтерияПредприятия` | Имя конфигурации (Метаданные.Имя) |
| `version` | `3.0.191.41` | Версия конфигурации (Метаданные.Версия) |
| `computer` | `BUHPC-01` | Имя компьютера пользователя |
| `connection_string` | `Srvr="srv1c";Ref="buh_prod";` | Строка подключения к базе 1С |

### 2. Чат подключается к Centrifugo и отправляет hello

```
Lyra-Chat.epf ──► WebSocket ──► Centrifugo (общий адрес:порт)
```

Первое сообщение в канал:

```json
{
  "type": "hello",
  "configuration": "БухгалтерияПредприятия",
  "version": "3.0.191.41",
  "computer": "BUHPC-01",
  "connection_string": "Srvr=\"srv1c\";Ref=\"buh_prod\";"
}
```

### 3. Роутер обрабатывает hello

```
Centrifugo ──► ЕХТ_Лира_Роутер
```

Роутер выполняет цепочку:

```
hello
  │
  ├─► MDM: найти или создать базу по (configuration + connection_string + computer)
  │         → идентификатор базы в НСИ
  │
  ├─► MDM: авторизация пользователя / базы
  │         → ОК / Отказ
  │
  ├─► Биллинг: проверить баланс
  │         → ОК / Баланс исчерпан
  │
  ├─► Vega: выбрать инстанс по (configuration + version)
  │         → порт MCP-сервера Vega для этой конфигурации
  │
  └─► Ответ клиенту: hello_ack
```

### 4. Клиент получает hello_ack

```json
{
  "type": "hello_ack",
  "session_id": "uuid-сессии",
  "database_id": "uuid-базы-в-НСИ",
  "status": "ok"
}
```

Или отказ:

```json
{
  "type": "hello_ack",
  "status": "error",
  "error": "Баланс исчерпан. Пополните на сайте."
}
```

## Отправка сообщения: полный цикл

### 5. Пользователь пишет вопрос в чате

```json
{
  "type": "chat",
  "session_id": "uuid-сессии",
  "content": "Почему у контрагента Ромашка не заполнен КПП?"
}
```

### 6. Модератор проверяет входящее сообщение

```
Сообщение пользователя ──► ЕХТ_Лира_Модератор ──► Модель (быстрая, дешёвая — Haiku или локальная)
                                                      │
                                                      ├─► Запрещённое → Предупреждение пользователю
                                                      ├─► Рискованное → Сообщение + приписка → Claude
                                                      └─► Обычное     → Сообщение без изменений → Claude
```

Три результата:

- **Блокировка** — prompt injection, вопросы об устройстве → пользователь получает предупреждение
- **Напоминание** — рискованный вопрос ("расскажи о себе", "какая ты модель?") → модератор добавляет к сообщению приписку: "При ответе придерживайся легенды"
- **ОК** — обычный вопрос, проходит без изменений

Если заблокировано:

```json
{
  "type": "moderator_warning",
  "message": "Я не могу отвечать на вопросы о внутреннем устройстве системы."
}
```

Соблюдение легенды ("Лира — AI-ассистент") обеспечивается двумя уровнями:

1. **Системный промпт** Claude (основная линия)
2. **Модератор** — контекстные напоминания на входе при рискованных вопросах

### 7. Роутер запускает адаптер (если ещё не запущен для сессии)

```
ЕХТ_Лира_Роутер ──► ЕХТ_СтдИО ──► adapter.exe
    параметры запуска:
      --system-prompt <промпт>
      --mcp-config <конфиг с Vega + mcp-1c-docs>
      --allowed-tools <список>
      --vega-port <порт для конфигурации клиента>
      --session-id <uuid>
```

### 8. Адаптер передаёт вопрос в Claude, стримит ответ

```
Адаптер ──► Claude
              │
              ├── thinking (размышления)
              ├── text (ответ по буквам)
              ├── tool_use (MCP-вызов)
              │     ├── Vega: search_metadata, search_code
              │     └── exec: v8_query, v8_eval → через Centrifugo → 1С клиента → результат
              └── result (финальный ответ)
```

### 9. Стриминг ответа клиенту

```
Claude ──► Адаптер ──► ЕХТ_Лира_Роутер ──► Centrifugo ──► Lyra-Chat.epf
                                                               │
                                                               ▼
                                                          Чат пользователя
                                                          (по буквам, realtime)
```

Проверка исходящих не нужна — легенда обеспечивается системным промптом + напоминаниями модератора на входе (шаг 6).

### 10. exec-команды (MCP → 1С клиента)

Когда Claude нужны данные из базы пользователя:

```
Claude: tool_use v8_query("ВЫБРАТЬ КПП ИЗ Справочник.Контрагенты WHERE ...")
    │
    ▼
Адаптер ──► Роутер ──► Centrifugo ──► Lyra-Chat.epf (база пользователя)
                                            │
                                            ▼
                                      Выполнить запрос на сервере 1С
                                            │
                                            ▼
Lyra-Chat.epf ──► Centrifugo ──► Роутер ──► Адаптер ──► Claude
    результат: [{КПП: "", ЮрФизЛицо: "ФизЛицо"}]
```

## Идентификация базы в НСИ (MDM)

Из hello формируется уникальный идентификатор базы:

```
configuration + connection_string + computer
     │                 │                │
     ▼                 ▼                ▼
"БухгалтерияПредприятия" + "Srvr=srv1c;Ref=buh_prod;" + "BUHPC-01"
                    │
                    ▼
            Хеш → идентификатор базы в НСИ
```

Это позволяет:
- Различать одну и ту же конфигурацию на разных серверах
- Различать тестовую и рабочую базу на одном сервере
- Привязывать базу к тарифу, балансу, истории запросов
- Выбирать правильный Vega-инстанс (по конфигурации + версии)

## Переподключение

Если клиент отключился и подключился обратно:

```json
{
  "type": "hello",
  "session_id": "uuid-предыдущей-сессии",
  "configuration": "БухгалтерияПредприятия",
  "version": "3.0.191.41",
  "computer": "BUHPC-01",
  "connection_string": "Srvr=\"srv1c\";Ref=\"buh_prod\";"
}
```

Роутер находит живую сессию адаптера по session_id, переподключает канал Centrifugo. Claude сохраняет контекст диалога.

## Компоненты и их роли (сводка)

| Компонент | Роль | Где живёт |
|-----------|------|-----------|
| **Lyra-Chat.epf** | UI чата, WebSocket-клиент, exec-команды | База пользователя |
| **Centrifugo** | WebSocket-транспорт, каналы, переподключение | Сервер Lyra |
| **ЕХТ_Лира_Роутер** | Бизнес-логика: авторизация, маршрутизация, запуск адаптеров | Серверная база 1С |
| **ЕХТ_Лира_Модератор** | Проверка сообщений перед отправкой в Claude | Серверная база 1С |
| **MDM** | НСИ: базы, пользователи, конфигурации | Серверная база 1С |
| **Биллинг** | Баланс, тарификация, учёт запросов | Серверная база 1С |
| **Адаптер (exe)** | Запуск Claude с параметрами, стриминг, MCP | Сервер Lyra |
| **Vega** | Метаданные конфигураций 1С, MCP-сервер | Сервер Lyra |
| **mcp-1c-docs** | Документация 1С, примеры, справочник языка | Сервер Lyra |
